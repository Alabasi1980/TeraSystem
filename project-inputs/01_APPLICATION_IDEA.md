# Application Idea — نظام إدارة طلبات المواد (MRMS)

## 1. Short Description
نظام داخلي لإدارة طلبات المواد (قرطاسية ومكتبية) داخل الشركة.
الموظف يقدم طلباً (من كتالوج أو إدخال يدوي)، المدير يوافق، والإدارة العليا تعتمد عند الحاجة، ثم يُصرف من المخزون أو يُشترى.

## 2. Problem the Application Solves
- تنظيم عملية طلب المواد وإلغاء المعاملات الورقية.
- توثيق مسار الموافقات الشفاف (من طلب → موافقة → تنفيذ).
- تمكين الموظفين من متابعة حالة طلباتهم.
- التحكم بالمخزون ومنع تكرار الطلبات على مواد غير متوفرة.
- توفير تقارير دقيقة للإدارة لتحليل المصروفات.

## 3. Expected Users
1. **موظف (Employee)** — يقدم طلبات المواد، يتابع حالة طلباته.
2. **مدير (Manager)** — يوافق/يرفض طلبات موظفيه.
3. **إدارة عليا (Senior Management)** — اعتماد إضافي أو إشراف عام.
4. **مسؤول النظام (Admin)** — إدارة المستخدمين، المواد، المخزون، الأقسام، الإعدادات.

## 4. Main Workflows
1. **تقديم طلب**: يختار الموظف مواد من الكتالوج أو يكتب وصفاً يدوياً ← يرفع مرفقات (صور، PDF) ← يرسل الطلب.
2. **الموافقة (هرمية)**: المدير المباشر يوافق أو يرفض ← (إن لزم) الإدارة العليا تعتمد.
3. **التنفيذ**: صرف المواد من المخزون إن كانت متوفرة، أو تحويل الطلب للشراء.
4. **إدارة المواد**: إضافة/تعديل/حذف المواد في الكتالوج مع الكميات المخزونة.
5. **إدارة المستخدمين والأقسام**: إنشاء المستخدمين، تحديد أدوارهم (Employee, Manager, Senior Management, Admin)، ربطهم بالأقسام.
6. **التقارير والإحصائيات**: عدد الطلبات، حالة كل طلب، إجمالي التكلفة، أكثر المواد طلباً، طلبات كل قسم وكل موظف، مصروفات شهرية.
7. **الإشعارات**: داخل التطبيق (In-App)، بريد إلكتروني، إشعارات PWA للجوال.

## 5. Expected Modules or Screens
1. **تسجيل الدخول** — نظام صلاحيات كامل (Roles-based)
2. **لوحة التحكم (Dashboard)** — ملخص سريع للطلبات والإحصائيات لكل دور
3. **إدارة الطلبات** — قائمة الطلبات، تقديم طلب جديد، تفاصيل الطلب، حالة الطلب
4. **كتالوج المواد** — قائمة المواد مع الكميات، إضافة/تعديل/حذف مواد
5. **إدارة المخزون** — متابعة الكميات، تنبيه عند انخفاض المخزون
6. **الموافقات** — قائمة طلبات بانتظار موافقتي (خاصة بالمدير والإدارة العليا)
7. **إدارة المستخدمين والأقسام** — Admin only
8. **التقارير والإحصائيات** — رسوم بيانية، جداول، تصدير PDF/Excel
9. **الإعدادات** — إعدادات عامة للتطبيق

## 6. Expected Outputs
- تأكيد تقديم الطلب (عرض + إشعار)
- إشعارات بريد إلكتروني عند تغير حالة الطلب
- تقارير PDF/Excel قابلة للتصدير
- سجل كامل لجميع الطلبات والموافقات

## 7. MVP Scope

> **ملاحظة**: هذا التصنيف مبني على تحليل Tera حسب `MVP_DEFINITION_PROTOCOL.md`. المستخدم قد يغيّر التصنيف، لكن Tera لا يضع ميزة تلقائيًا في Core MVP لمجرد أن المستخدم اختارها أثناء الاكتشاف.

### Core MVP (Phase 1A) — أقل نسخة قابلة للاستخدام
- تسجيل دخول مع صلاحيات (Employee, Manager, Senior Management, Admin)
- إدارة المستخدمين والأقسام الأساسية
- كتالوج المواد (إضافة، عرض، تعديل أساسي)
- تقديم طلب مواد (اختيار من كتالوج + إدخال يدوي)
- موافقة المدير (قبول/رفض)
- موافقة الإدارة العليا (عند الحاجة)
- حالات الطلب (قيد الانتظار، موافق، مرفوض، منفذ)
- واجهة متجاوبة (Responsive Web) — الجوال والمتصفح
- دعم اللغة العربية والإنجليزية مع RTL/LTR

### Extended MVP (Phase 1B) — يُبنى بعد Core مباشرة
- إرفاق مرفقات مع الطلب (صور، PDF)
- إشعارات داخل التطبيق (In-App)
- لوحة تحكم أساسية (Dashboard) مع ملخص سريع
- تقارير أساسية (عدد الطلبات، الحالات، إجمالي التكلفة)

## 8. Phase 2
- إدارة المخزون (متابعة الكميات، صرف، إضافة)
- تنبيهات انخفاض المخزون
- إشعارات البريد الإلكتروني
- تصدير Excel/PDF
- Audit Log بسيط

## 9. Phase 3
- PWA + Push Notifications للجوال
- موافقات حسب قيمة الطلب (توجيه ذكي)
- تقارير وتحليلات متقدمة (رسوم بيانية، مقارنات)
- سجل تدقيق تفصيلي (Audit Log)

## 10. Later / Enterprise
- تطبيق جوال منفصل (Native)
- تكامل مع ERP / أنظمة محاسبية
- ميزانية ملزمة للأقسام (Hard Budget)

## 11. Out of Scope
- الدفع الإلكتروني أو المشتريات الخارجية (فقط إدارة الطلب والموافقة)

## 12. Open Questions
- هل يحتاج كل دور إلى صلاحيات دقيقة (مثلاً: Manager يوافق حتى مبلغ معين فقط)؟ — قيد المناقشة
- هل هناك حاجة لسير عمل موافقة متعدد المستويات بحسب قيمة الطلب؟
- هل هناك نظام للموافقات الجماعية (مثلاً: لجنة مشتريات)؟
- ما هي طريقة إرسال البريد الإلكتروني (SMTP داخلي أم خدمة خارجية)؟
