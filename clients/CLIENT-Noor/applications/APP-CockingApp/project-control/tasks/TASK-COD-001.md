## 1. Objective

Create the initial CockingApp application scaffold using Next.js + TypeScript and install Prisma dependencies, while keeping the first task strictly limited to safe foundation work.

This task must create only a basic Prisma configuration:

```text
generator client
datasource db
```

No Prisma models, migrations, database apply commands, UI components, API routes, or Auth are allowed in this task.

---

## 2. Approved Execution Scope

### Included

- Scaffold a Next.js app in:
  ```text
  clients/CLIENT-Noor/applications/APP-CockingApp/cocking-app/
  ```
- Use TypeScript, ESLint, App Router, `src/` directory, npm.
- Do **not** include Tailwind in scaffold.
- Install Prisma dependencies only:
  - `prisma`
  - `@prisma/client`
- Manually create `prisma/schema.prisma` with only generator + datasource.
- Create `.env.example` with redacted placeholders only.
- Create `src/lib/prisma.ts` singleton without real DB connection test.
- Ensure root layout uses Arabic/RTL basics if created by scaffold.

### Explicitly Excluded

- Prisma models / entities / enums.
- `npx prisma init` if it creates `.env`.
- `npx prisma generate`.
- `npx prisma db push`.
- `npx prisma migrate *`.
- Seed data.
- API routes.
- Auth/JWT/Cookies/Middleware.
- UI components / Tailwind / design implementation.
- Real `.env` or any real secret.
- Git commit/push.
