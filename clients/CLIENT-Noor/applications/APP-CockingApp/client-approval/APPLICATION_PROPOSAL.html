عرض تنفيذ وتطوير تطبيق CockingApp

يوضح هذا العرض الفهم التنفيذي لتطبيق الوصفات المقترح، النطاق المتوقع للنسخة الأولى، المتطلبات الأساسية، الاقتراحات التطويرية المبنية على أبحاث السوق، ومراحل التنفيذ المقترحة.

الملخص التنفيذي

Executive Summary

CockingApp هو تطبيق ويب متكامل لإدارة وعرض وصفات الطبخ، يجمع بين لوحة تحكم (Dashboard) قوية للنظام (Admin) وواجهة عرض أنيقة للزوار. يتميز بنظام مكونات منفصل (Ingredient Management) يتيح تعريف المكونات ككيانات مستقلة وربطها بالوصفات — بدلاً من الكتابة الحرة التقليدية، مما يتيح تنظيمًا احترافيًا وإعادة استخدام المكونات بذكاء. يدعم التطبيق الصور، فيديوهات YouTube، التعليقات العامة، وتحميل PDF للوصفات.

فهم المشروع والحل المقترح

Problem / Solution

المشكلة الحالية

  • • الحاجة إلى تنظيم الوصفات بشكل احترافي في مكان واحد
  • • مشكلة الكتابة الحرة للمقادير تؤدي إلى صعوبة التنظيم والتكرار وعدم الاتساق
  • • عدم وجود قناة موحدة لعرض الوصفات مع الصور والفيديو والتفاعل مع الجمهور
  • • صعوبة مشاركة الوصفات وتحميلها بتنسيق منظم (PDF)

الحل المقترح

  • • نظام مكونات منفصل — تعريف المكونات وإعادة استخدامها عبر الوصفات
  • • لوحة تحكم كاملة لإدارة المحتوى (وصفات، تصنيفات، مكونات، تعليقات)
  • • واجهة عرض عامة أنيقة مع دعم الصور والفيديو والتعليقات وتحميل PDF
  • • تصميم عصري (Claude Design System) مع تجربة مستخدم سلسة

المستخدمون والأدوار

Users / Roles

المستخدمون المستهدفون

Admin (نور): مدير المحتوى الأساسي — يدخل لوحة التحكم، يدير التصنيفات، المكونات، الوصفات، الصور، الفيديو، والتعليقات.

زوار عام: جمهور التطبيق — يتصفحون الوصفات حسب التصنيفات، يشاهدون التفاصيل والصور والفيديو، يعلقون، ويحملون PDF بدون تسجيل دخول.

الأدوار والصلاحيات

Admin: صلاحية كاملة — إدارة التصنيفات، المكونات، الوصفات، الصور والفيديو، التعليقات (موافقة/حذف)، وإعدادات النظام.

زوار: تصفح عام، بحث، تعليق (بدون حساب)، تحميل PDF، حفظ مفضلة (محلي).

نطاق العمل — النسخة الأولى

Phase 1 Scope

النطاق الأساسي — مرحلة 1A

  • لوحة تحكم Admin — Dashboard رئيسي مع إحصائيات
  • إدارة التصنيفات (إضافة / تعديل / حذف / ترتيب)
  • إدارة المكونات ككيانات منفصلة مع وحدات قياس (كوب، ملعقة، غرام، كغم، مل، حبة)
  • إدارة الوصفات — إنشاء/تعديل مع ربط المكونات بالكميات
  • رفع الصور لكل وصفة (معرض صور)
  • إضافة روابط فيديو YouTube لكل وصفة
  • إضافة خطوات التحضير (نصوص منسقة)
  • عرض عام للوصفات — صفحة رئيسية، تصنيفات، بحث
  • صفحة وصفة مفصلة (مقادير، خطوات، صور، فيديو، تعليقات)
  • تحميل PDF لكل وصفة
  • نظام التعليقات العامة (بدون حساب)
  • إدارة التعليقات من لوحة التحكم (موافقة/حذف)

خارج النطاق الحالي

  • تطبيق جوال (Native Mobile App)
  • تكامل مع أجهزة المطبخ الذكية
  • متجر إلكتروني لبيع المنتجات أو المكونات
  • نظام اشتراكات مدفوعة
  • استيراد فيديوهات من TikTok / Instagram مباشرة
  • نظام أدوار متعدد (Admin فقط حاليًا)
  • الذكاء الاصطناعي لتوليد الوصفات (AI Recipe Generation)

مقترحات تحسينية — مبنية على أبحاث السوق

Market Research Suggestions

بناءً على تحليل أفضل تطبيقات الوصفات العالمية والعربية لعام 2025-2026 (مثل Samsung Food, Cookpad, Paprika, Mealime, Yummly, SideChef، وسمسم، وأطيب طبخة)، نقترح الميزات التطويرية التالية:

موصى به — ضمن النسخة الأولى مباشر

  • قائمة مشتريات ذكية — تحويل مكونات الوصفات إلى لائحة تسوق منظمة حسب الأقسام
    (مصدر: Samsung Food, Cookpad, سمسم)
  • مقياس الوصفات الديناميكي (Recipe Scaling) — تغيير عدد الأشخاص والمقادير تتغير تلقائياً
    (مصدر: Paprika, Mealime)
  • وقت التحضير والطهي + بحث بالوقت — عرض وقت التحضير والطهي لكل وصفة مع فلترة حسب الوقت
    (مصدر: كل التطبيقات الرائدة)
  • مفضلة / حفظ الوصفات — للزوار حفظ الوصفات المفضلة محلياً بدون حساب
    (مصدر: Paprika, Cookpad)

المرحلة الثانية لاحقاً

  • معلومات غذائية — حساب السعرات والمغذيات (بروتين، كارب، دهون) لكل حصة
    (مصدر: Samsung Food, EatThisMuch)
  • خطة وجبات أسبوعية — جدولة الوجبات للأسبوع مع إشعارات
    (مصدر: Mealime, سمسم)
  • الطبخ بدون إنترنت (Offline) — مشاهدة الوصفات المحفوظة بدون نت
    (مصدر: Paprika — ميزة قوية جداً)
  • بحث بالمكونات المتوفرة — أدخل مكوناتك والتطبيق يقترح وصفات مناسبة
    (مصدر: SuperCook, Yummly, Cookpad)
  • نظام تقييم ونجوم — تقييم الزوار للوصفات بجانب التعليقات
    (مصدر: Cookpad)
  • تقليل هدر الطعام — اقتراح وصفات بناءً على المكونات الموجودة حالياً
    (مصدر: Crumb, EatThisMuch)

مراحل متقدمة مستقبلاً

  • اقتراح وصفات بالذكاء الاصطناعي — "ماذا أطبخ اليوم؟" بناءً على المكونات المتاحة
  • بدائل المكونات الذكية — "استبدال البيض" يعطي بدائل مناسبة
  • ميزانية التسوق — تتبع مصروفات المشتريات واقتراح بدائل أقل تكلفة
  • لائحة مشتريات متعاونة — مشاركة قائمة التسوق مع العائلة في الوقت الفعلي
  • تطبيق جوال / PWA — تجربة أفضل على الجوال

مقترحات للمناقشة

• التحكم الصوتي — أوامر صوتية أثناء الطبخ (SideChef, Yummly) - مناسب للمستخدمين المتقدمين مستقبلاً

• استيراد الوصفات من المواقع — إمكانية لصق رابط وصفة واستيرادها تلقائياً (Paprika Feature) — مناسب للمرحلة الثانية

• التكامل مع واتساب — مشاركة الوصفات عبر واتساب بضغطة واحدة

المتطلبات الأساسية

Requirements

المتطلبات الوظيفية

• إدارة كاملة للتصنيفات والمكونات والوصفات
• واجهة عامة للعرض والتصفح والبحث
• تعليقات عامة مع إدارة من Admin
• تحميل PDF لكل وصفة
• رفع صور وإضافة روابط YouTube

المتطلبات التقنية

• Next.js (App Router) + TypeScript
• PostgreSQL + Prisma ORM
• Tailwind CSS
• استضافة On-premise
• RTL (عربي) بالكامل

متطلبات البيانات

• المكونات — كيانات منفصلة مع وحدات قياس
• الوصفات — ربط بالمكونات والكميات
• التصنيفات — هيكل هرمي
• الصور والفيديو — روابط/مرفقات
• التعليقات — مع حالة الموافقة

متطلبات التصميم

• Claude Design System (getdesign.md)
• ألوان: كريمي (#faf9f5) + مرجاني (#cc785c) + كحلي (#181715)
• خطوط: Copernicus-like (serif) + Inter (sans)
• RTL بالكامل
• متجاوب مع جميع الشاشات

متطلبات الأمان

• حماية لوحة التحكم بكلمة مرور
• فصل تام بين Admin والواجهة العامة
• التحقق من التعليقات (موافقة مسبقة)
• منع رفع الملفات الضارة

متطلبات التشغيل

• استضافة On-premise (على سيرفر العميل)
• Docker-ready (اختياري)
• قاعدة بيانات PostgreSQL
• دعم النسخ الاحتياطي للبيانات

القيمة المتوقعة من التطبيق

Expected Value

تنظيم احترافي

نظام المكونات المنفصل يضمن الاتساق عبر جميع الوصفات ويمنع التكرار والكتابة العشوائية — سهولة البحث والتصفية حسب المكونات.

تجربة زوار ممتازة

واجهة عرض أنيقة مع دعم الوسائط المتعددة، تحميل PDF، تعليقات، ومقياس ديناميكي للوصفات — كل ما يحتاجه الزائر في مكان واحد.

قابلية التوسع

بنية تقنية مرنة (Next.js + Prisma) تسمح بإضافة الميزات المستقبلية بسهولة مثل خطة الوجبات، المعلومات الغذائية، والتطبيق الجوال.

الافتراضات والنقاط المفتوحة

Assumptions
المجال الافتراض الحالة
التقنية Next.js + Prisma + PostgreSQL — المكدس الموصى به مؤكد
التصميم Claude design system من getdesign.md — بألوانه وخطوطه مؤكد
اللغة العربية (RTL) فقط — لا حاجة للإنجليزية حالياً مؤكد
الاستضافة On-premise — على سيرفر العميل (تحديد المواصفات لاحقاً) يحتاج تأكيد
الفيديو روابط YouTube فقط — لا رفع مباشر للفيديو مؤكد
التصنيفات تصنيفات طبخ تقليدية (يحددها Admin لاحقاً) يحتاج تأكيد
التعليقات عامة بدون حساب — مع موافقة مسبقة من Admin يحتاج تأكيد
المقترحات التحسينية قائمة مشتيات، Scaling، وقت التحضير — مقترحة ضمن المرحلة الأولى بانتظار موافقة العميل

خارطة التنفيذ المقترحة

Delivery Roadmap
1

المرحلة الأولى — Core MVP

النواة الأساسية: لوحة تحكم Admin كاملة (تصنيفات، مكونات، وصفات)، واجهة عرض عامة، صور، فيديو YouTube، تعليقات، تحميل PDF، قائمة مشتريات، مقياس الوصفات، وقت التحضير، مفضلة.

2

المرحلة الثانية — Extended MVP

معلومات غذائية (سعرات حرارية)، خطة وجبات أسبوعية، بحث بالمكونات المتوفرة، تقييم ونجوم، تقليل هدر الطعام، طبخ بدون إنترنت، استيراد وصفات من المواقع.

3

مراحل لاحقة — Future

اقتراح وصفات بالذكاء الاصطناعي، بدائل المكونات الذكية، ميزانية التسوق، تطبيق جوال / PWA، مشاركة القوائم، تحكم صوتي، تكامل واتساب.
