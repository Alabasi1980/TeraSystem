# SCOPE_SUMMARY — نطاق العمل

> **نسخة محدّثة:** 2026-07-05 — إزالة قطع الغيار من المرحلة الأولى حسب قرار Majed. متوافقة مع SCP-038.

---

## داخل النطاق (المرحلة الأولى — MVP)

### 1. إدارة العملاء
- سجل موحد للعملاء (الاسم، رقم الهاتف، العنوان، ملاحظات)
- إضافة/تعديل/حذف عميل
- بحث عن عميل
- عرض تاريخ طلبات العميل

### 2. إدارة طلبات الصيانة
- إنشاء طلب صيانة جديد برقم موحد (تلقائي)
- تسجيل: العميل، الجهاز، الماركة، المشكلة، تاريخ الاستلام
- تعيين فني مسؤول للطلب
- تحديث حالة الطلب عبر 10 حالات (انظر أدناه)
- إضافة ملاحظات ونتائج الفحص
- تسجيل التكلفة المتوقعة والتكلفة الفعلية
- إضافة القطع المستخدمة في الطلب (إدخال نصي يدوي — بدون نظام مخزون)
- رفع صورة إن لزم (اختياري)

### 3. حالات الطلب
1. طلب جديد
2. محدد له موعد
3. قيد الفحص
4. بانتظار موافقة العميل (مع تسجيل الموافقة/الرفض)
5. قيد الصيانة
6. بانتظار قطعة غيار
7. جاهز للتسليم
8. تم التسليم
9. مغلق
10. ملغي

### 4. إدارة الفنيين
- قائمة الفنيين
- إضافة/تعديل/حذف فني
- عرض الطلبات المفتوحة لكل فني
- لوحة الفني: مشاهدة طلباته المسندة فقط

### 5. الصلاحيات والأدوار
- **مدير**: صلاحية كاملة (مستخدمين، تقارير، إيرادات، كل الطلبات)
- **استقبال**: إضافة عملاء وطلبات، متابعة حالة، لا يعدّل الأسعار أو التقارير المالية
- **فني**: مشاهدة طلباته المسندة فقط، تحديث التشخيص والحالة الفنية
- **محاسب**: مشاهدة المدفوعات والتقارير المالية، لا يغيّر تفاصيل الصيانة

### 6. تسجيل التكلفة والدفع
- تسجيل تكلفة الصيانة (أجرة + تكلفة القطع المدخلة يدوياً)
- تسجيل حالة الدفع: (غير مدفوع / مدفوع جزئياً / مدفوع بالكامل)
- إمكانية إصدار إيصال بسيط

### 7. الإشعارات (داخل التطبيق) — اختياري ضمن الميزانية
> ملاحظة: هذه الميزة مصنفة كـ ◉ Optional في FEATURE_LIST.md. إذا سمحت الميزانية، تُضاف:
- إشعار عند تغيير حالة الطلب
- إشعار عند إسناد طلب لفني
- إشعار عند تأخر طلب عن المدة المتوقعة
- إشعار عند جهوزية الجهاز للتسليم

### 8. التقارير الأساسية
- تقرير الطلبات المتأخرة
- تقرير يومي بعدد الطلبات (جديد/مغلق/قيد التنفيذ)
- تقرير شهري بالإيرادات
- فلترة حسب: التاريخ، الفني، حالة الطلب

### 9. Dashboard
- عرض ملخص سريع: عدد الطلبات المفتوحة، المتأخرة، المنجزة اليوم
- آخر الطلبات المضافة
- إشعارات سريعة

## خارج النطاق (المرحلة الأولى)

- ❌ تطبيق جوال منفصل (نظام ويب متجاوب يغطي الاحتياج)
- ❌ واتساب API تلقائي
- ❌ بوابات دفع إلكتروني
- ❌ حسابات دخول للعملاء
- ❌ فواتير مفصلة (إيصال بسيط فقط)
- ❌ إدارة المشتريات والموردين
- ❌ طلبات شراء قطع غيار
- ❌ تتبع GPS للفنيين
- ❌ دعم متعدد اللغات

## مؤجل للمرحلة الثانية (Phase 2)

| النطاق | التفاصيل |
|--------|---------|
| 🛠️ **قطع الغيار ونظام المخزون الأساسي** | تعريف قطع الغيار (الاسم، الكمية، سعر التكلفة، سعر البيع)، تسجيل الكمية المتوفرة، اختيار القطع المستخدمة داخل طلب الصيانة مع إنقاص الكمية تلقائياً، تقرير بسيط بالقطع المنخفضة |
| 📊 **تقارير متقدمة** | تقرير أداء الفنيين (إن لم يُدرج في Phase 1)، تقرير القطع الأكثر استخداماً، رسوم بيانية |
| 🤖 **واتساب API تلقائي** | إشعارات واتساب/SMS تلقائية للعملاء |
| 💳 **بوابة دفع إلكتروني** | تكامل مع بوابة دفع خارجية |
| 📦 **إدارة مخزون متقدمة** | فواتير مشتريات، موردين، طلبات شراء |
| 📋 **إدارة عقود صيانة** | عقود صيانة دورية للعملاء |

## القيود

- اللغة: العربية (RTL) فقط
- المنصة: Web App (متجاوب مع الجوال للاستخدام الأساسي)
- عدد المستخدمين الأساسي: ~7
- الميزانية: 1,500 JOD حد أقصى صارم (أي توسعات = عرض سعر منفصل)

## الافتراضات

- سير العمل الحالي (موثق أعلاه) هو الأساس
- الموافقة على التكلفة تتم يدوياً عبر التطبيق (العميل غير متصل)
- الصور ترفق بملاحظات الطلب يدوياً
- الاستضافة ستكون سحابية مع خيار مناسب للتكلفة — تكلفة منفصلة عن التطوير
- قطع الغيار تُسجل يدوياً في حقل النص في المرحلة الأولى (بدون نظام مخزون)
