# خطة العمل المقترحة — نظام إدارة ورشة التصنيع المعدني

| المعلومة | القيمة |
|:---------|:-------|
| **العميل** | شركة سعيد للمقاولات |
| **المصنع** | مصنع تصنيع هياكل معدنية وحديد — يخدم مشاريع الشركة |
| **إعداد** | TeraClientEngagementAgent (مُستشار) + Domain Expert Agent |
| **التاريخ** | 2026-07-11 |
| **الحالة** | مسودة — تحتاج اعتماد العميل قبل البدء |

---

## 1. ملخص تنفيذي

**الوضع الحالي:** المصنع يستقبل طلبات تصنيع من مشاريع الشركة عبر الهاتف والواتساب والإيميل، ويُنفذها بدون توثيق أو إثباتات. لا يوجد سجل لأوامر التصنيع، لا تتبع للمراحل، لا فحص جودة موثق، ولا توثيق للتسليم.

**المشكلة:** غياب التوثيق يؤدي إلى:
- عدم معرفة أين كل طلب في أي وقت
- عدم وجود سجل مرجعي عند حدوث خطأ
- صعوبة في تحسين الإنتاجية لعدم وجود بيانات تاريخية
- اعتماد كامل على ذاكرة الأفراد

**الحل المقترح:** نظام إلكتروني خفيف (Manufacturing Execution System — MES) مخصص لإدارة أوامر التصنيع وتتبعها من الاستلام حتى التسليم، يُطبق على 3 مراحل متدرجة.

---

## 2. ما تم إنجازه — مرحلة البحث والتحليل

| النشاط | النتيجة |
|:-------|:--------|
| بحث متخصص في أفضل ممارسات إدارة ورش التصنيع المعدني | ✅ تقرير استخباراتي (Domain Intelligence Report) |
| تحليل مجال التصنيع وتصنيفه | ✅ Job Shop / Project-based Manufacturing |
| مقارنة الحلول الموجودة في السوق | ✅ 5 حلول تم تحليلها |
| إعداد نموذج بيانات أولي وهيكل النظام | ✅ 8 وحدات، 18 شاشة، نموذج بيانات كامل |

**الخلاصة الرئيسية من البحث:**

النظام المخصص الخفيف هو الأنسب لهذه الورشة. الحلول الجاهزة (EZIIL, Tekla PowerFab, Odoo) إما غالية جداً، أو معقدة، أو لا تناسب ورشة داخلية بهذا الحجم.

## 3. تصور النظام — نظرة عامة

### ما سيفعله النظام — Phase 1 (المرحلة الأساسية)

```
┌─────────────────────────────────────────────────────────────┐
```

### دورة حياة الأمر

```
استلام طلب → إنشاء أمر في النظام (برسمة ومواصفات)
→ تحديد مراحل التصنيع المطلوبة
→ البدء في أول مرحلة (عبر تابلت/جوال)
→ تتبع التقدم عبر المراحل حتى اكتمال التصنيع
→ فحص الجودة (Phase 2)
→ تسليم وتوثيق الاستلام (Phase 2)
→ إغلاق الأمر
```

## 4. خريطة المراحل (Phased Rollout)

### 🟢 المرحلة 1 — النظام الأساسي (Core MVP)
**المدة المقترحة:** 6-8 أسابيع

| الوحدة | الوظائف الرئيسية |
|:-------|:-----------------|
| **إدارة أوامر التصنيع** | إنشاء أمر، تحديد حالة، ربط المشروع، توثيق الإجراءات |
| **تتبع الإنتاج** | لوحة الورشة، Start/Pause/Complete لكل مرحلة، تسجيل الوقت |
| **إدارة المخططات** | رقم رسمة، رقم مراجعة، رفع ملف (PDF/DXF) |
| **المستخدمون والصلاحيات** | 3 أدوار: مدير، مهندس، عامل |
| **التقارير الأساسية** | لوحة بيانات، تقرير حالة الطلبات، سجل إجراءات |
| **الإعدادات** | إدارة المراحل مرنة (إضافة/تعديل/حذف) |

**شرط القبول:** 10 أوامر تصنيع حقيقية تُدار وتُتتبع كاملة في النظام.

### 🔵 المرحلة 2 — الجودة والتسليم
**المدة المقترحة:** 4-6 أسابيع بعد Phase 1

| الوحدة | الوظائف الرئيسية |
|:-------|:-----------------|
| **مراقبة الجودة** | فحص Pass/Fail، توثيق العيوب، أوامر إعادة العمل |
| **التسليم** | قائمة جاهزة للتسليم، تأكيد استلام، توثيق |
| **تتبع القطع الفردية** | Piece-level tracking لكل قطعة في الأمر |
| **الجدولة** | أولويات، تواريخ استلام، Gantt بسيط |
| **تقارير موسعة** | WIP Age، On-Time Delivery، Rework Rate |

**شرط القبول:** 50 أمراً مكتملاً مع بيانات QC وتسليم.

### 🟡 المرحلة 3 — المواد والتحليلات (اختياري)
**المدة المقترحة:** 4-6 أسابيع

| الوحدة | الوظائف الرئيسية |
|:-------|:-----------------|
| **قوائم المواد (BOM)** | مواد لكل أمر، ربط بالمخزون |
| **المخطط مقابل الفعلي** | مقارنة ساعات تقدير مع فعلية |
| **لوحة مؤشرات الأداء** | تحليلات متقدمة، اختناقات، إنتاجية |

## 5. لماذا النظام المخصص وليس حلولاً جاهزة؟

| الحل | التقييم | السبب |
|:-----|:--------|:------|
| **EZIIL** | ❌ غير مناسب | نظام متخصص لمصانع تجارية — غالي (آلاف سنوياً)، إنجليزي، لا يخدم ورشة داخلية |
| **Tekla PowerFab** | ❌ غير مناسب | معقد جداً، سعري باهظ، مصمم لمقاولي حديد كبار — ميزات زائدة 80% |
| **Steel Projects** | ❌ غير مناسب | ERP كامل — 70% من ميزاته خارج حاجة الورشة الحالية |
| **Odoo MES** | 🟡 ممكن | مفتوح المصدر لكنه ERP عام يحتاج تخصيصاً كبيراً للتصنيع المعدني |
| **نظام مخصص (مقترح)** | ✅ الأنسب | خفيف، مخصص للورشة، عربي 100%، يكلف لمرة واحدة، مرن كامل |

## 6. المخاطر المتوقعة وكيف سنتعامل معها

| المخاطرة | مستوى الخطورة | استراتيجية التعامل |
|:---------|:-------------:|:-------------------|
| **مقاومة العمال للنظام** | 🔴 عالي | واجهة بسيطة (3 أزرار)، تدريب عملي، "بطل تغيير" في الورشة |
| **بيانات غير دقيقة** | 🟡 متوسط | سجل إجراءات (Audit Log)، مراجعات دورية |
| **تعقيد تنوع المنتجات** | 🟡 متوسط | مرونة في مسارات التصنيع — كل منتج له مساره الخاص |
| **أوامر عالقة (Zombie Orders)** | 🟡 متوسط | تقرير WIP Age يومي — تنبيه للطلبات الخاملة |
| **فقدان بيانات أثناء الانتقال** | 🟡 متوسط | فترة تشغيل موازي أسبوعين — النظام + الواتساب معاً |

## 7. نصائح للنجاح

- **نبدأ صغير:** 5 طلبات حقيقية تدخل النظام أول أسبوعين
- **واجهة الورشة بسيطة جداً:** زر أخضر (ابدأ)، أصفر (أوقف مؤقتاً)، أزرق (أكمل)
- **"بطل تغيير":** شخص واحد متحمس في الورشة يُدرّب الباقين
- **تحديثات أسبوعية:** نسمع ملاحظات العمال ونحسّن باستمرار
- **فترة تشغيل موازي:** أسبوعان — النظام والطرق القديمة معاً — لطمأنة الجميع
