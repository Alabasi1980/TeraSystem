# CLIENT_INTAKE.md — شركة الحوت للمقاولات

> تاريخ الإنشاء: 2026-07-06
> المصدر: محادثة مباشرة مع Majed

---

## 1. معلومات العميل الأساسية

| الحقل | القيمة |
|:------|:-------|
| اسم العميل | شركة الحوت للمقاولات |
| نوع العميل | شركة مقاولات |
| حجم الشركة | من أضخم شركات المقاولات في الأردن |
| المجال | مقاولات عامة — تشمل مشتريات متنوعة (مباشرة، غير مباشرة، مقاولي باطن) |

## 2. المشروع

| الحقل | القيمة |
|:------|:-------|
| نوع المشروع | **دراسة استشارية — توثيق وتحليل** (وليس تطوير برمجي) |
| النطاق | قسم المشتريات فقط — البداية |
| الهدف | إنتاج دراسة كاملة لمشتريات المقاولات مطابقة بنسبة 90% لأفضل الممارسات العالمية (ERP standards) |
| المستخدم النهائي | الدراسة ستُسلّم لشركة برمجيات متخصصة لبناء ERP كامل لاحقاً |
| المنتج النهائي | حزمة توثيق متكاملة، مرجع واحد موحّد، بدون ذكر مصادر ERP محددة |

## 3. المنهجية

- **لا نعتمد على عمليات الحوت الحالية كأساس** — بل نبدأ من أفضل الممارسات العالمية
- المصادر: SAP MM، Oracle Procurement، Microsoft Dynamics 365 Supply Chain (دون الإفصاح عنها)
- مدير المشتريات في الحوت: مراجع ومشرف — لتأكيد الاقتراحات وتزويدنا بأي تفاصيل خاصة بالشركة
- المنتج النهائي: دراسة متكاملة تُغطي كل صغيرة وكبيرة في المشتريات

## 4. الأطراف المعنية

| الطرف | الدور |
|:------|:------|
| Majed | المالك — صاحب القرار، المشرف العام |
| TCEA (مُستشار) | بناء الدراسة بالكامل — البحث، التحليل، التوثيق، الصياغة |
| مدير المشتريات (الحوت) | مراجع ومشرف — معلومات إضافية، تأكيد الاقتراحات |
| شركة برمجيات (لاحقاً) | المستلم النهائي للدراسة — لبناء ERP |

## 5. المبادئ التوجيهية

- الدراسة بالعربية (مع احتمالية حاجة مصطلحات إنجليزية تقنية)
- تغطية شاملة — لا يُفقد أي جزء حتى لو كان صغيراً
- لا يُذكر أي مصدر ERP (لا SAP، لا Oracle، لا Microsoft)
- المنتج النهائي مرجع واحد موحّد لا يحتوي مقارنات أو خيارات

## 6. Notes

- ستحتاج الدراسة لمراجعة من مدير المشتريات قبل اعتمادها
- هذا مشروع من نوع "معقد" (ربما يميل للغامض) — يتطلب بحثاً مكثفاً عبر الويب
- المسار: `studies/` (وليس `applications/`) نظراً لطبيعة المنتج الاستشارية
