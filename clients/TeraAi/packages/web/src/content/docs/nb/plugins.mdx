**npm-plugins** installeres automatisk ved hjelp av Bun ved oppstart. Pakker og deres avhengigheter er bufret i `~/.cache/opencode/node_modules/`.

**Lokale plugins** lastes direkte fra plugin-katalogen. For ШЈВҐ bruke eksterne pakker, mШЈВҐ du opprette en `package.json` i konfigurasjonskatalogen din (se [Dependencies](#dependencies)), eller publisere plugin-en til npm og [legg den til i konfigurasjonen din](/docs/config#plugins).

Plugins lastes inn fra alle kilder og alle kroker kjШЈВёres i rekkefШЈВёlge. LastrekkefШЈВёlgen er:

3. Global plugin-katalog (`~/.config/opencode/plugins/`)
4. Prosjektpluginkatalog (`.opencode/plugins/`)

Dupliserte npm-pakker med samme navn og versjon lastes inn ШЈВ©n gang. Imidlertid lastes en lokal plugin og en npm plugin med lignende navn begge separat.

## Opprette en plugin

En plugin er en **JavaScript/TypeScript-modul** som eksporterer en eller flere plugin-moduler
funksjoner. Hver funksjon mottar et kontekstobjekt og returnerer et krokobjekt.

---

### Avhengigheter

Lokale plugins og tilpassede verktШЈВёy kan bruke eksterne npm-pakker. Legg til en `package.json` til konfigurasjonskatalogen med avhengighetene du trenger.

```json title=".opencode/package.json"

  "dependencies": {
    "shescape": "^2.1.0"
  }
}
```

OpenCode kjШЈВёrer `bun install` ved oppstart for ШЈВҐ installere disse. Programtilleggene og verktШЈВёyene dine kan deretter importere dem.

```ts title=".opencode/plugins/my-plugin.ts"

export const MyPlugin = async (ctx) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        output.args.command = escape(output.args.command)
      }
    },
  }
}
```

### Grunnleggende struktur

```js title=".opencode/plugins/example.js"
export const MyPlugin = async ({ project, client, $, directory, worktree }) => {
  console.log("Plugin initialized!")

  return {
    // Hook implementations go here
  }
}
```

Plugin-funksjonen mottar:

- `project`: Gjeldende prosjektinformasjon.
- `directory`: Gjeldende arbeidskatalog.
- `worktree`: Git-arbeidstrebanen.
- `client`: En OpenCode SDK klient for samhandling med AI.
- `$`: Buns [shell API](https://bun.com/docs/runtime/shell) for ШЈВҐ utfШЈВёre kommandoer.

### Egendefinerte verktШЈВёy

### Komprimerings-hooks
