## 2. المشكلة الحالية

- الموظفون يرسلون طلبات الصيانة عبر WhatsApp أو اتصالات هاتفية
- يؤدي ذلك إلى **ضياع بعض الطلبات**
- **عدم وضوح الحالة** لكل طلب (هل تم الاستلام؟ هل قيد التنفيذ؟ هل اكتمل؟)

---

## 3. الحل المطلوب

تطبيق ويب داخلي (Internal Web App) يتيح:

| المستخدم | المهام |
|----------|--------|
| **الموظف** (~25 مستخدم) | تسجيل طلب صيانة ← متابعة حالته ← عرض طلباته السابقة |
| **مسؤول الصيانة** (2 مستخدم) | عرض جميع الطلبات ← تحديث الحالة ← إضافة تعليقات ← إغلاق الطلب |
| **المدير الإداري** (فارس) | لوحة متابعة (Dashboard) + تقارير + إشراف عام |

## 5. النطاق المبدئي

### داخل النطاق (In Scope — MVP)
- [x] تسجيل دخول (username/password) مع 3 أدوار صلاحيات
- [x] نموذج إضافة طلب صيانة (قسم، نوع الصيانة، وصف، أولوية)
- [x] عرض قائمة الطلبات مع تصفية حسب الحالة/التاريخ
- [x] حالات الطلب: جديد ← قيد التنفيذ ← مكتمل ← مغلق
- [x] تحديث الحالة + إضافة تعليق من مسؤول الصيانة
- [x] Dashboard إداري: عدد الطلبات الجديدة، قيد التنفيذ، المكتملة، المتأخرة
- [x] عرض الموظف لطلباته السابقة
- [x] تصنيفات الصيانة: كهرباء، تكييف، أثاث، أجهزة مكتبية، شبكة/إنترنت، أخرى
- [x] حقل القسم/الموقع في الطلب

### خارج النطاق (Out of Scope — MVP)
- [ ] إرفاق صور/مرفقات ← يُؤجل
- [ ] إشعارات بريد إلكتروني ← يُؤجل
- [ ] تطبيق موبايل ← يُؤجل (يكفي Web Responsive)
- [ ] بحث متقدم ← يُؤجل
- [ ] تكاملات خارجية ← غير مطلوبة حالياً
- [ ] إدارة المستخدمين (تسجيل ذاتي) ← يُدار من قبل المسؤول
