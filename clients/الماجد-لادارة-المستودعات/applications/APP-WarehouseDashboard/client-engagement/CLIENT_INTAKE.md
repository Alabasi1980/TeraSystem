## مكونات المشروع — بنية مزدوجة (API + Razor)

### المكون 1: WarehouseDashboard.API (ASP.NET Core Web API على .NET 8)
- **استخراج جداول من Oracle** عبر ODP.NET / Oracle.ManagedDataAccess
- **تحويل وتسجيل البيانات في SQL Server**
- **المزامنة:**
  - حذف الموجود + إدخال الجديد (Full Refresh)
  - تلقائية بفترة محددة (TBD)
  - يدوية عبر زر في الـ Dashboard
- **Synchronization Logs** — تسجيل كل عملية مزامنة (وقت النجاح/الفشل، عدد السجلات، المدة)
- **Database Connection:** Oracle + SQL Server على نفس السيرفر المحلي

### المكون 2: WarehouseDashboard.Web (ASP.NET Core Razor Pages على .NET 8)
- **داشبورد تفاعلي** بـ ~20 بطاقة (Syncfusion)
- **Drill Down** — كل بطاقة لها مستويات قابلة للتكوين
- **لوحة تحكم ديناميكية (Admin Panel):**
  - صفحة منفصلة محمية بكلمة مرور
  - لا تُعرض في القائمة — يمكن الوصول فقط عبر الرابط المباشر
  - إنشاء/تعديل/حذف البطاقات
  - تحديد مصدر البيانات لكل بطاقة (SQL Query / View)
  - تحديد نوع العرض (Bar, Line, Pie, KPI, Table)
  - تحديد مستويات الـ Drill Down والربط
  - حفظ التكوين في قاعدة البيانات
- **الداشبورد يقرأ التكوين ويبني البطاقات ديناميكياً**
- **Phase 2 (مؤجل):** شاشات تعديل وإضافة بيانات + صلاحيات مستخدمين

## البنية التحتية

| العنصر | القيمة |
|--------|--------|
| **التكنولوجيا** | ASP.NET C# على .NET 8 (LTS) |
| **نمط الواجهة** | Razor Pages |
| **قاعدة بيانات المصدر** | Oracle (محلية على سيرفر الزبون) |
| **قاعدة بيانات الهدف** | SQL Server (محلية على نفس السيرفر) |
| **آلية الاتصال بـ Oracle** | ODP.NET / Oracle.ManagedDataAccess |
| **آلية الاتصال بـ SQL Server** | Entity Framework Core / ADO.NET |
| **بيئة النشر** | IIS داخلي على سيرفر الزبون |
| **مكتبة الـ Dashboard** | Syncfusion (Community License — مجاني) |
| **Fallback Dashboard** | Chart.js + D3.js (إذا لم تنطبق رخصة Syncfusion) |
