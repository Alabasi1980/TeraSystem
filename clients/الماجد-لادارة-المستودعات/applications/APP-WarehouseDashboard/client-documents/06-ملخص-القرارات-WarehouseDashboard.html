السادة المحترمون،

تحية طيبة وبعد،

في هذا الموجز عرض لأبرز القرارات الفنية والإدارية التي تمّ اتخاذها لمشروع WarehouseDashboard، مع مبرراتها وحالتها. جميع القرارات المدرجة هنا معتمدة من الفريق المشترك.

القرارات التقنية الرئيسية

القرار 1: اختيار .NET 8 (LTS)

  • التوصية: .NET 8 — أحدث إطار عمل من Microsoft مع دعم طويل الأمد (LTS) حتى 2026.
  • المبرر: أداء أعلى، مكتبات أحدث، توافق ممتاز مع بيئة Windows و IIS.
  • الحالة: ✅ معتمد.

القرار 2: نمط الواجهة — Razor Pages

  • التوصية: ASP.NET Core Razor Pages لتطبيق Dashboard.
  • المبرر: بساطة التطوير، تكامل مباشر مع Syncfusion + Entity Framework.
  • الحالة: ✅ معتمد.

القرار 3: مكتبة الداشبورد — Syncfusion

  • التوصية: Syncfusion Essential Studio مع Community License.
  • المبرر: أكثر من 1,600 مكوّن، دعم ممتاز لـ .NET 8، خاصية Drill Down مدمجة.
  • البديل: Chart.js + D3.js (لم يكن ضرورياً).
  • الحالة: ✅ معتمد — الرخصة متوفرة (مفتاح UnlockKey موجود).

القرار 4: الاتصال بقاعدة Oracle — ODP.NET

  • التوصية: Oracle.ManagedDataAccess (ODP.NET Managed Driver).
  • المبرر: اتصال مباشر بـ Oracle دون الحاجة إلى تثبيت Oracle Client.
  • الحالة: ✅ معتمد.

القرار 5: طريقة الإدخال المجمّع — ADO.NET SqlBulkCopy

  • التوصية: استخدام ADO.NET مع SqlBulkCopy للإدخال المجمّع بدلاً من EF Core.
  • المبرر: أسرع بـ 3-5 مرات من EF Core عند التعامل مع كميات كبيرة من البيانات.
  • EF Core: يُستخدم فقط لجداول التكوين وسجلات المزامنة (البيانات الصغيرة).
  • الحالة: ✅ معتمد — قرار استراتيجي.

القرار 6: جاهزية Incremental Sync

  • التوصية: تصميم محرك المزامنة ليدعم Incremental Sync من البداية، حتى لو لم يُفعّل في Phase 1.
  • المبرر: يمنع الحاجة إلى إعادة كتابة المحرك بالكامل مستقبلاً عند الحاجة للتحديث التزايدي.
  • الحالة: ✅ معتمد — قرار استراتيجي.

القرار 7: اختبار Oracle في وقت مبكر

  • التوصية: اختبار الاتصال بقاعدة Oracle خلال أول 3 أيام من بدء التنفيذ.
  • المبرر: يقلّل مخاطر اكتشاف مشاكل الاتصال في وقت متأخر.
  • الحالة: ✅ معتمد.

القرار 8: أمان Admin Panel — كلمة مرور فقط

  • التوصية: حماية Admin Panel بكلمة مرور بسيطة + تشفير (BCrypt/PBKDF2).
  • الوصول: صفحة غير مرئية في القائمة — وصول عبر الرابط المباشر فقط.
  • الحالة: ✅ معتمد — مع إمكانية تطوير صلاحيات أكثر في Phase 2.

القرارات الإدارية والتنظيمية

القرار 9: طريقة التسعير — Time & Material

  • السعر: 4 JOD لكل ساعة.
  • النطاق المقدر: 430 — 625 ساعة = 1,720 — 2,500 JOD.
  • الحالة: ✅ معتمد.

القرار 10: الأولوية — API أولاً ثم Dashboard

  • الترتيب: بناء WarehouseDashboard.API أولاً، ثم WarehouseDashboard.Web.
  • المبرر: Dashboard يعتمد على البيانات التي يوفّرها API.
  • الحالة: ✅ معتمد.

القرار 11: المدة الزمنية — بدون deadline

  • العمل بالسرعة الطبيعية مع الالتزام بالجودة العالية.
  • الحالة: ✅ معتمد.

القرار 12: الميزانية — مفتوحة

  • التكلفة ليست العامل الأوّل — التركيز على الجودة.
  • الحالة: ✅ معتمد.

القرار 13: الدعم بعد التسليم — صيانة + تطوير

  • إصلاح أخطاء + تطوير مستمر حسب الاحتياجات المستقبلية.
  • الحالة: ✅ معتمد.
