السادة المحترمون،

تحية طيبة وبعد،

في هذا الموجز نقدّم تصوّراً واضحاً لكيفية عمل تطبيق WarehouseDashboard من منظور المستخدم، مع شرح مسارات العمل الرئيسية وواجهات الشاشات المتوقعة.

أوّلاً: مسار المزامنة (Data Sync Workflow)

يبدأ عمل النظام من قاعدة Oracle حيث يتم استخراج البيانات حسب جدول زمني محدّد (مزامنة تلقائية) أو عند الطلب (مزامنة يدوية). تمرّ عملية المزامنة بالخطوات التالية:

  • بدء المزامنة: إمّا تلقائياً عبر مؤقّت خلفي (Background Service) أو يدوياً عبر النقر على زر "مزامنة الآن" في Dashboard.
  • استخراج البيانات: يقوم محرك المزامنة بقراءة الجداول المحدّدة من Oracle عبر ODP.NET.
  • تحويل البيانات: تحويل أنواع البيانات من Oracle إلى صيغ SQL Server.
  • الإدخال المجمّع: حذف جميع البيانات الموجودة في الجداول الهدف في SQL Server، ثم إدخال البيانات الجديدة دفعة واحدة باستخدام تقنية SqlBulkCopy عالية الأداء.
  • تسجيل العملية: تسجيل العملية في جدول السجلات (وقت البدء، وقت الانتهاء، الحالة، عدد السجلات، المدة، أي أخطاء).

ثانياً: مسار عرض Dashboard (Dashboard Workflow)

عند دخول المستخدم إلى صفحة Dashboard الرئيسية:

  • يقوم النظام بقراءة تكوين البطاقات من قاعدة بيانات SQL Server.
  • ينفّذ الاستعلام المرتبط بكل بطاقة للحصول على البيانات.
  • يعرض البطاقات في تخطيط شبكي متجاوب مع دعم 6 أنواع مختلفة من البطاقات.
  • يظهر في أعلى الصفحة شريط حالة المزامنة يعرض آخر وقت مزامنة وحالة النظام.
  • يمكن للمستخدم النقر على زر المزامنة اليدوية لبدء مزامنة فورية.

ثالثاً: خاصية التعمق في البيانات (Drill Down)

عند النقر على أي بطاقة، يمكن للمستخدم التعمق في بياناتها عبر مستويات متعددة قابلة للتكوين. على سبيل المثال:

  • بطاقة إجمالي المشتريات ← النقر عليها يعرض تفاصيل المشتريات حسب المورّد ← النقر على مورّد يعرض فواتيره.
  • يظهر شريط التنقل (Breadcrumb) لتوضيح المستوى الحالي وإمكانية العودة للخلف.
  • كل بطاقة لها مستوياتها المستقلة التي يمكن تكوينها من Admin Panel.

رابعاً: مسار Admin Panel (Config Workflow)

Admin Panel هو لوحة تحكم منفصلة تُستخدم لإدارة النظام دون التأثير على واجهة المستخدم:

  • الدخول: يتم عبر الرابط المباشر مع إدخال كلمة مرور. الصفحة غير مرئية في قوائم التنقل.
  • إدارة البطاقات: يمكن إضافة بطاقة جديدة، تعديل بطاقة موجودة، أو حذف بطاقة.
  • تكوين البطاقة: عند إنشاء أو تعديل بطاقة، يتاح للمسؤول تحديد:
    • الاسم والعنوان الظاهر
    • مصدر البيانات (SQL Query أو View)
    • نوع العرض (Bar, Line, Pie, KPI, Table, Gauge)
    • مستويات Drill Down (استعلام خاص بكل مستوى)
    • الحجم والموقع في الشبكة
  • اختبار الاستعلام: يمكن اختبار أي استعلام قبل حفظه للتأكد من صحّة النتائج.
  • الحفظ: يُحفظ التكوين في قاعدة البيانات وتتحدّث Dashboard فوراً.

خامساً: أنواع البطاقات المدعومة

  • Bar Chart: مقارنة بين فئات مختلفة (مثل: المشتريات لكل شهر)
  • Line Chart: عرض الاتجاهات الزمنية (مثل: حركة المخزون خلال عام)
  • Pie Chart: توزيع نسبي (مثل: توزيع المشتريات حسب المورّد)
  • KPI Card: رقم رئيسي مع مؤشر لأداء (مثل: إجمالي قيمة المخزون + نسبة التغيّر)
  • Data Table: جدول تفصيلي يعرض البيانات كاملة
  • Gauge: مؤشر أداء بصري (مثل: نسبة إشغال المستودع)

سادساً: وصف الشاشات

Dashboard الرئيسية:

  • شريط علوي: اسم النظام + شعار + شريط حالة المزامنة (آخر مزامنة + مؤشر حالة + زر مزامنة)
  • منطقة البطاقات: تخطيط شبكي يعرض جميع البطاقات المكوّنة مع إمكانية التمرير
  • كل بطاقة تعرض عنوانها + الرسم البياني/البيانات المخصّصة

Admin Panel:

  • صفحة دخول: حقل كلمة مرور
  • لوحة التحكم: قائمة البطاقات مع خيارات (إضافة، تعديل، حذف)
  • محرر البطاقة: نموذج يحتوي على جميع خيارات التكوين
  • صفحة اختبار الاستعلام: محرر SQL + زر تنفيذ + عرض النتائج

سجلات المزامنة:

  • جدول يعرض جميع عمليات المزامنة مع تفاصيلها (التاريخ، الحالة، عدد السجلات، المدة)
  • إمكانية تصفية السجلات وعرض تفاصيل كل عملية
