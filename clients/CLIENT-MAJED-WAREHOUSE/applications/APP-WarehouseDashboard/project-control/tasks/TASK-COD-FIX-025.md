## 1. Objective

Fix the missing anti-forgery token on the TableMappings admin page hidden form, which prevents the wizard save from reaching the server handler (same issue as Card Builder TASK-COD-FIX-021).

Auditor finding F-001 from `QUAUD-TABLEMAPPINGS-001-2026-07-18-001.md` confirmed that the hidden POST form at line 897 of `Index.cshtml` lacks `@Html.AntiForgeryToken()`, and the inline forms (Toggle, PreviewSchema, ApplySchema, Delete) also require verification.
