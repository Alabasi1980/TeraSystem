- التأكد من أن جميع وظائف التطبيق تعمل بشكل صحيح

- التأكد من نجاح المزامنة التلقائية واليدوية
- التأكد من أمان لوحة الإدارة
- التأكد من أداء التطبيق ضمن الحدود المقبولة
- التأكد من التعامل السليم مع حالات الخطأ

| **المختبر (Majed)** | تنفيذ الاختبارات وتسجيل النتائج |

| # | الاختبار | الوصف | خطوات التنفيذ | النتيجة المتوقعة | النتيجة |
|---|---------|-------|--------------|-----------------|---------|
| **أ1** | اختبار اتصال Oracle | التحقق من الاتصال بقاعدة Oracle | `POST /api/sync/test-oracle` | رد `200 OK` مع معلومات الاتصال (Server Version, User, SID) | ☐ |
| **أ2** | اختبار اتصال SQL Server | التحقق من الاتصال بقاعدة SQL Server | `POST /api/sync/test-sql` | رد `200 OK` مع معلومات الاتصال (Server, Database, Version) | ☐ |
| **أ3** | اختبار قراءة بيانات Oracle | التحقق من قراءة بيانات فعلية من Oracle | `GET /api/sync/oracle-data` | رد `200 OK` مع JSON يحتوي على بيانات جداول Oracle (مصفوفة من الكائنات) | ☐ |

  ```

  ```
- **أ2:**
  ```
  curl -X POST http://10.10.1.1/api/sync/test-sql
  ```
- **أ3:**
  ```
  curl -X GET http://10.10.1.1/api/sync/oracle-data
  ```
- **أ4:** افتح SQL Server Management Studio وتحقق من جداول `TableMapping` في قاعدة `WarehouseDashboard`

## 5. الفئة ب: اختبارات المزامنة

> **الهدف:** التأكد من عمل محرك المزامنة (يدوياً وتلقائياً) وتسجيل السجلات.

| # | الاختبار | الوصف | خطوات التنفيذ | النتيجة المتوقعة | النتيجة |
|---|---------|-------|--------------|-----------------|---------|
| **ب1** | تشغيل المزامنة يدوياً | تفعيل دورة مزامنة يدوية | `POST /api/sync/trigger` مع `{"mode": "full"}` | رد `202 Accepted` مع `syncId` و `status: "running"` أو `"queued"` | ☐ |
| **ب2** | الحالة بعد التشغيل | مراقبة حالة المزامنة بعد التشغيل | `GET /api/sync/status` (انتظر بضع ثوانٍ بعد ب1) | `isRunning: true` مع `progress` يแสดง نسبة الإنجاز | ☐ |
| **ب3** | اكتمال المزامنة | انتظار اكتمال دورة المزامنة | انتظر اكتمال التشغيل ثم `GET /api/sync/status` | `isRunning: false`، `lastSync.status: "Success"`، `progress.percent: 100` | ☐ |
| **ب4** | وصول البيانات لـ SQL Server | التحقق من وصول البيانات المُزامّنة | فتح SQL Server Management Studio وفحص جداول البيانات | البيانات موجودة في الجداول المُزامّنة | ☐ |
| **ب5** | سجلات المزامنة | التحقق من تسجيل دورة المزامنة | `GET /api/sync/logs` | رد `200 OK` مع مصفوفة سجلات تحتوي على الدورة التي تم تنفيذها | ☐ |
| **ب6** | سجلات المزامنة — تفاصيل | مراجعة تفاصيل السجل | `GET /api/sync/logs?page=1&pageSize=10` | السجل يحتوي على `syncId`، `startTime`، `endTime`، `status`، `recordCount`، `durationMs` | ☐ |
| **ب7** | المزامنة التلقائية | تفعيل المزامنة التلقائية وانتظار التشغيل | من Admin Panel → SyncSettings → فعّل Auto-Sync → انتظر الفاصل الزمني (30 دقيقة افتراضياً) | المزامنة تبدأ تلقائياً بعد انتهاء الفترة | ☐ |
| **ب8** | منع التشغيل المتداخل | محاولة تشغيل مزامنة أثناء تشغيل حالية | شغّل مزامنة يدوية ← أثناء التشغيل، شغّل مزامنة أخرى | المزامنة الثانية تُرفض بحالة `skipped` أو `409 Conflict` مع `errorCode: SYNC_ALREADY_RUNNING` | ☐ |
| **ب9** | اختبار وضع Incremental | محاولة استخدام الوضع التفاضلي (غير مفعّل) | `POST /api/sync/trigger` مع `{"mode": "incremental"}` | رد `501 Not Implemented` مع `errorCode: INCREMENTAL_NOT_ENABLED` | ☐ |
| **ب10** | سجلات المزامنة — فلترة بالحالة | فلترة السجلات حسب الحالة | `GET /api/sync/logs?status=Success` | تظهر فقط سجلات النجاح | ☐ |

### ملاحظات تنفيذ:

- **ب1:** استخدم `curl`:
  ```
  curl -X POST http://10.10.1.1/api/sync/trigger -H "Content-Type: application/json" -d "{\"mode\":\"full\"}"
  ```
- **ب7:** لتفعيل المزامنة التلقائية من Admin Panel:
  1. سجّل الدخول إلى `/admin-secure-panel`
  2. افتح صفحة `SyncSettings`
  3. فعّل خيار Auto-Sync
  4. احفظ الإعدادات
  5. انتظر الفترة المحددة

## 6. الفئة ج: اختبارات لوحة الإدارة (Admin Panel)

> **الهدف:** التأكد من عمل لوحة الإدارة بالكامل — تسجيل الدخول، إدارة البطاقات، إعدادات المزامنة.

### 6.1 المصادقة (Authentication)

| # | الاختبار | الوصف | خطوات التنفيذ | النتيجة المتوقعة | النتيجة |
|---|---------|-------|--------------|-----------------|---------|
| **ج1** | تسجيل الدخول بنجاح | تسجيل الدخول بكلمة مرور صحيحة | افتح `/admin-secure-panel` ← أدخلباسورد صحيح ← اضغط دخول | يتم التحويل لصفحة لوحة الإدارة مع عرض المحتوى | ☐ |
| **ج2** | تسجيل الخروج | تسجيل الخروج من الجلسة | سجّل الدخول ← اضغط زر "خروج" | يتم مسح الجلسة والتحويل لصفحة تسجيل الدخول | ☐ |

## 7. الفئة د: اختبارات لوحة التحكم (Dashboard)

> **الهدف:** التأكد من عرض البيانات بشكل صحيح وعمل جميع العناصر التفاعلية.

### 7.1 تحميل الصفحة وعرض البيانات

| # | الاختبار | الوصف | خطوات التنفيذ | النتيجة المتوقعة | النتيجة |
