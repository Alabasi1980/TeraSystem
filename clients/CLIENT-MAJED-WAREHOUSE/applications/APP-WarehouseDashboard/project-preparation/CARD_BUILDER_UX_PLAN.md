## 1. المشكلة الحالية

- صفحة `Cards/Create` تعرض العديد من حقول الإدخال النصية.
- مناسبة للمطور، غير مناسبة للمستخدم العادي.
- لا توجد معاينة مباشرة.
- لا توجد قوالب جاهزة.

## 2. المبدأ التصميمي

**“من البساطة إلى العمق”**
- 90% من المستخدمين يحتاجون خطوات قليلة + اختيارات.
- 10% فقط يحتاجون إعدادات متقدمة (مخفية).

## 3. آلية العمل المقترحة

### 3.1 Wizard من 4 خطوات
1. **النوع** — اختيار Tile: KPI / Bar / Line / Pie / Table / Gauge
2. **المصدر** — اختيار من قائمة:
   - بطاقة جاهزة (Template)
   - Query محفوظ
   - Oracle table/view
   - SQL query (متقدم فقط)
3. **الحقول الأساسية** — فقط:
   - العنوان
   - اسم العرض
   - مصدر البيانات
   - القياس/الحقل (من قائمة منسدلة)
4. **الشكل** — Tiles للحجم/الموضع/الألوان/التحديث

### 3.2 معاينة مباشرة (Live Preview)
- يظهر على الجانب بطاقة حقيقية كما ستكون.
- تتحدث فورًا عند أي تغيير.
- زر **حفظ / حفظ وإضافة أخرى**.

### 3.3 قوالب جاهزة (Templates)
- بطاقات مبدئية شائعة (مخزنة مسبقًا).
- المستخدم يبدأ منها بدل الصفر.

### 3.4 نسخ بطاقة موجودة
- زر “Clone” يفتح المنشئ بإعدادات البطاقة المحددة.

### 3.5 خيارات متقدمة (Accordion)
- مخفية افتراضيًا.
- SQL يدوي، فلاتر، Drill-down، تسميات مخصصة.

## 4. مكونات الواجهة

| العنصر | الوصف |
|---|---|
| Step Indicator | شريط خطوات أعلى الصفحة |
| Type Picker | بطاقات صغيرة قابلة للنقر (أيقونة + اسم) |
| Source Selector | قائمة منسدلة مع بحث |
| Field Mapper | سحب وإفلات أو اختيار من قائمة |
| Visual Editor | أزرار/سحّابات للحجم والموضع واللون |
| Live Preview Pane | عرض البطاقة فعليًا |
| Advanced Accordion | قسم قابل للطي للخيارات العميقة |

## 5. سهولة الماوس/اللمس

- كل الاختيارات بـ **نقر واحد**.
- لا حقول نصية إلا للضروري (العنوان، اسم العرض).
- تلميحات (Tooltips) على كل خطوة.
- أزرار واضحة: التالي/السابق/إنهاء.
