السادة المحترمون،

تحية طيبة وبعد،

نستعرض في هذا الموجز آلية تنفيذ المرحلة الأولى (MVP) من مشروع WarehouseDashboard، وفقاً للمنهجية المعتمدة من مؤسسة تيرا للنظم (Teranoo) لضمان الجودة والشفافية في جميع مراحل التطوير.

أوّلاً: منهجية التنفيذ

تعتمد آلية التنفيذ على منهجية Iterative Development (تطوير تكراري) مع تسليمات مرحلية. يتم بناء API (محرك المزامنة) أولاً ثم Dashboard، مع اختبار كل مرحلة قبل الانتقال إلى التي تليها، وذلك وفق الأولوية المُعتمدة من شركتكم.

ثانياً: مراحل التنفيذ

المرحلة 1 — التهيئة والإعداد (الأسبوع 1)

  • إنشاء حل .NET 8 بمشروعين (API + Razor)
  • إعداد قاعدة بيانات SQL Server (جداول التكوين، السجلات، البيانات)
  • تجهيز بيئة IIS للنشر
  • اختبار الاتصال بقاعدة Oracle (خلال أول 3 أيام)

المرحلة 2 — بناء API (محرك المزامنة) — الأسبوع 2 إلى 4

  • تكوين الاتصال بقاعدة Oracle باستخدام ODP.NET
  • بناء محرك الاستخراج مع تحويل أنواع البيانات
  • تطبيق آلية Full Refresh (حذف + إدخال مجمّع باستخدام ADO.NET SqlBulkCopy)
  • إعداد محرك المزامنة لدعم Incremental Sync مستقبلاً (جاهزية استراتيجية)
  • بناء نقاط الـ API: /trigger، /status، /logs، /config
  • تسجيل عمليات المزامنة مع تتبّع الحالة والمدة
  • اختبار المزامنة اليدوية والتلقائية

المرحلة 3 — بناء Dashboard (الأسبوع 5 إلى 8)

  • إعداد التخطيط الأساسي مع Syncfusion
  • بناء آلية عرض البطاقات الديناميكية (قراءة التكوين من DB)
  • تطبيق 6 أنواع من البطاقات (Bar, Line, Pie, KPI, Table, Gauge)
  • بناء خاصية Drill Down مع Breadcrumb Navigation
  • تطبيق الهوية البصرية (Blue Theme — 11 لوناً)
  • إضافة شريط حالة المزامنة وزر المزامنة اليدوية
  • ضبط التجاوب مع جميع أحجام الشاشات

المرحلة 4 — بناء Admin Panel (الأسبوع 9 إلى 11)

  • صفحة دخول محمية بكلمة مرور (وصل عبر الرابط المباشر فقط)
  • واجهة CRUD للبطاقات (إنشاء، تعديل، حذف)
  • تكوين مصادر البيانات (SQL Query / View)
  • اختيار نوع العرض وتحديد مستويات Drill Down
  • اختبار الاستعلام قبل الحفظ (Query Tester)
  • تكوين تخطيط البطاقات (الحجم والترتيب)

المرحلة 5 — التكامل والاختبار (الأسبوع 12 إلى 13)

  • اختبار التكامل بين API و Dashboard
  • اختبار سيناريوهات المزامنة (يدوي، تلقائي، أخطاء)
  • اختبار Drill Down عبر مستويات متعددة
  • اختبار Admin Panel (CRUD + حفظ التكوين)
  • اختبار الأداء مع بيانات فعلية

المرحلة 6 — النشر والتسليم (الأسبوع 14)

  • نشر API و Dashboard على IIS في خادم الشركة
  • تسليم كود المصدر مع تعليمات التشغيل الأساسية
  • تدريب فريق العميل على استخدام النظام

ثالثاً: متطلبات من العميل

  • توفير بيانات الاتصال بقاعدة Oracle (Server, Port, SID, User, Password)
  • تحديد الجداول المطلوب مزامنتها من Oracle (أثناء التنفيذ)
  • الوصول إلى سيرفر IIS للنشر أو توفير صلاحيات الإعداد
  • التوفر للإجابة عن الاستفسارات أثناء مرحلة اكتشاف الجداول

رابعاً: معايير القبول والتسليم

  • اكتمال جميع الميزات المدرجة في Phase 1 وفقاً لقائمة الميزات (FEATURE_LIST.md)
  • نجاح المزامنة التلقائية واليدوية مع تسجيل العمليات
  • عرض Dashboard كامل البطاقات مع Drill Down يعمل بشكل صحيح
  • Admin Panel يتيح إنشاء وتعديل وحذف البطاقات وحفظ التكوين
  • النظام يعمل بشكل مستقر على IIS
