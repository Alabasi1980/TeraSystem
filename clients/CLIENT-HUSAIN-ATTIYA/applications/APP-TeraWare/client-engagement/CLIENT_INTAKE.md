## مكونات المشروع — بنية مباشرة (بدون API Layer)

### المكون 1: TeraWare.Desktop (WPF + WebView2)
- **اتصال مباشر بـ Oracle** عبر `Oracle.ManagedDataAccess`
- **واجهة عرض وتعديل المواد:** تعرض بيانات المواد من Oracle مباشرة مع:
  - **تعديل فردي وجماعي** (✅ UPDATE فقط على جدول المواد، ❌ لا إضافة/حذف)
  - **قراءة فقط** لبقية الجداول (المستودعات، المشاريع، المستخدمين، حركات السندات)
- **استعلامات مباشرة من Oracle:**
  - بيانات المواد (جدول ديناميكي + تصدير Excel + فلاتر)
  - أرصدة المواد لكل مستودع (جدول ديناميكي + تصدير Excel + فلاتر)
  - الحركات اليومية (جدول ديناميكي + تصدير Excel + فلاتر)
- **المتصفح المخصص (WebView2):** يعرض Oracle APEX ERP مع JavaScript injection:
  - ملف JavaScript شامل يُحقن عند تشغيل المتصفح
  - تعديل لبلات العناصر (Labels)
  - إضافة أزرار مع أحداث
  - قراءة بيانات إضافية من Oracle
  - حفظ بيانات محددة من DOM إلى SQL Server
  - تصفح صفحات APEX عبر URL parameters

### المكون 2: TeraWare.Web (ASP.NET Core Razor Pages على IIS)
- **طلبات المواد:** إنشاء، تحضير، طباعة — متقدم مع متابعة
- يعمل من **الموبايل والمتصفح** (Responsive via Bootstrap)
- يُنشر على **IIS داخل موقع الشركة**
- **قاعدة بيانات خاصة: SQL Server** — يحفظ فيه طلبات المواد + إعدادات المتصفح
- **الربط مع Oracle:** حقل (رقم طلب المواد في Oracle) يُملأ يدوياً → المتابعة تتم داخل NatejSoft ERP
- **الإشعارات:** ⏳ مؤجلة (واتساب، بريد)

## قواعد الوصول إلى Oracle

| الجدول | نوع الوصول |
|--------|:----------:|
| المواد (Items) | ✅ قراءة + **UPDATE** (فردي وجماعي) |

| مجموعات المواد | ✅ قراءة فقط |
| حركات السندات (رئيسي + تفصيلي) | ✅ قراءة فقط |
| المستودعات | ✅ قراءة فقط |
| المشاريع | ✅ قراءة فقط |
