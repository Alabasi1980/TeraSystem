## 1. المشكلة

حالياً، `TeraClientEngagement.md §11` يقول صراحة:  
> "لم يُبنَ نظام تسعير كامل الآن. هذا القسم يحدد المبادئ فقط."

هذا يعني:
- لا آلية موحدة لتقدير الأسعار
- تقديرات غير متسقة بين المشاريع
- التطبيق الصغير قد يُسعّر مثل الكبير
- اعتماد كامل على تقدير يدوي

## 3. القرارات الاستراتيجية

| القرار | الاختيار |
|--------|---------|
| **طريقة التسعير** | Feature-Based + Complex Rubric (6 معايير) |
| **اشتقاق السعر** | ساعات متوقعة لمستوى Simple × سعر داخلي للساعة |
| **هامش الربح** | 20% تقدير أولي — يُعاد فحصه بعد أول مشروعين |
| **العملة** | دينار أردني (JOD) — لا تشمل ضرائب/رسوم/استضافة |
| **الصيانة** | 3 أشهر Bug Fix Warranty ← بدون اشتراك: يُسعّر لكل حالة |
| **الاستضافة** | الزبون يتحملها — Tera يوصي فقط |
| **خطة الدفع** | 50%-25%-25% كـ Default — قابلة للتعديل |
| **صلاحية السعر** | 14 يوماً (افتراضي) / 30 يوماً (للكبير) |

## 4. معادلة التسعير الداخلية

```
Feature Base Price (ساعات Simple × سعر الساعة الداخلي)
× Complexity Factor (1.0 / 1.5 / 2.5 / 3.5+)
+ Integration Costs
+ Risk Buffer (0% / 10% / 20%)
+ QA / Documentation Allowance
+ Margin (20%)
= Client Quote Price
```

## 5. Rubric تصنيف التعقيد (6 معايير × 4 مستويات)

| المعيار | 0点 | 1点 | 2点 | 3点 |
|---------|-----|-----|-----|-----|
| جداول البيانات | 0-1 | 2-3 | 4-6 | 7+ |
| منطق الأعمال | لا يوجد | شرط واحد | 2-3 شروط | Workflow معقد |
| الصلاحيات | دور واحد | 2-3 أدوار | 4-5 أدوار | 6+ أدوار |
| تكاملات API | 0 | 1 | 2 | 3+ |
| تقارير | لا يوجد | تقرير بسيط | رسوم بيانية | لوحة قيادة |
| إشعارات | 0 | بريد | بريد+SMS |实时 |

| النقاط | المستوى | المعامل |
|--------|---------|---------|
| 0-3 | **Simple** | 1.0x |
| 4-7 | **Medium** | 1.5x |
| 8-12 | **Complex** | 2.5x |
| 13+ | **Critical** | 3.5x أو خاص |

## 6. Rubric تصنيف Risk Buffer (5 معايير)

| المعيار | Low (0%) | Medium (+10%) | High (+20%) |
|---------|----------|---------------|-------------|
| وضوح المتطلبات | واضحة | عامة | غامضة |
| تقنية التطوير | مألوفة | جديدة نسبياً | غير مألوفة |
| تاريخ الزبون | سابق/موصى به | جديد بمعلوماته | جديد غير معروف |
| اعتماد API | 0 | 1-2 معروفة | 3+ |
| الجدول الزمني | 8+ أسابيع | 4-8 أسابيع | <4 أسابيع |

3+ معايير High → High Risk | 2 → Medium | غير ذلك → Low

## 7. قواعد الحماية المضمنة

| القاعدة | الغرض |
|---------|-------|
| **Minimum Project Price** | 500-700 JOD ويب / 1,200 JOD نظام |
| **Discovery/Paid Analysis** | 50-100 JOD أو 5% — يُخصم عند التعاقد |
| **6 تصنيفات Change Request** | Minor مجاني / Structural يُسعّر / Scope يُسعّر / إلخ |
| **المصفوفة داخلية** | عرض السعر للزبون مبسط — التفاصيل داخلية |
| **الأسعار لا تشمل** | ضرائب، رسوم حكومية، بوابات دفع، اشتراكات خارجية، استضافة |
| **TCEA يصدر Draft فقط** | Majed يعتمد السعر النهائي — لا عرض سعر دون اعتماد |
| **Post-Warranty Fallback** | بدون اشتراك صيانة: يُسعّر لكل حالة |
