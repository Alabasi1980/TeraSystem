# SYSTEM_CHANGE_PROPOSAL

## SCP-2026-07-07-077 — إنشاء عميل "مصمم واجهات" (UIDesignerAgent) بتخصص بصري عالٍ

---

### Request Type
New Agent Creation — Specialized UI/UX Visual Designer

### Requested By
TeraAgent + Majed — بعد فجوة عملية في مشروع Dashboard Prototype

---

### Problem

المنظومة الحالية تعاني من **فجوة في القدرة التنفيذية البصرية**:

| العميل | الدور | هل ينفذ واجهات؟ |
|--------|-------|:----------------:|
| TeraAgent | مدير مشروع — ينسّق ولا ينفذ | ❌ |
| ناقد (DesignReviewer) | مراجع بصري مستقل — **يراجع فقط** | ❌ |
| SoftwareDesigner | مصمم تقني — يكتب Technical Specs، لا يكتب كود | ❌ |
| general agent | مبرمج عام — ليس مختصاً بالجماليات والذوق البصري | ⚠️ بدون تخصص |
| DomainExpert | محلل مجال — ليس له علاقة بالواجهات | ❌ |

**النتيجة:** عند الحاجة إلى تصميم بصري فاخر (كما في مشروع Dashboard Prototype الحالي)، تضطر TeraAgent للتنفيذ يدوياً — وهذا غير صحيح إدارياً لأنه مدير وليس منفّذ، وتظهر النتيجة دون المستوى المطلوب (ناقد أكد أن التصميم ~55% فقط).

### Evidence

1. `dashboard-premium-prototype/` — بروتوتايب الداشبورد الحالي رفضه Majed لضعف المستوى البصري
2. تقرير DesignReviewer (ناقد) أكد 10 مشاكل بصرية حرجة ومتوسطة
3. لا يوجد عميل مختص بالتصميم البصري في فئة الـ `primary` أو `subagent`
4. تكرار المشكلة: نفس الفجوة ظهرت مع prototype PlayStation (اضطر Tera لتنفيذه بنفسه)

### Solution

إنشاء عميل **UIDesignerAgent (مصمم واجهات)** — وهو عميل منفّذ متخصص بالتصميم البصري فائق الجودة، يمتلك:
- أدواته الخاصة (webfetch للبحث عن تصاميم ومكتبات، Playwright للمعاينة)
- صلاحيات قراءة وكتابة وتنفيذ
- توجيه بالبحث عن أحدث التوجهات والقوالب قبل التنفيذ

| الحقل | القيمة |

---

#### 2. الهدف والغرض

تزويد منظومة Tera بقدرة **تنفيذ واجهات بصرية فاخرة ودقيقة** — تصميم وتنفيذ وإخراج نهائي بجودة احترافية عالية، مع القدرة على:

- **قراءة أي دليل تصميم** (getdesign.md، Figma Tokens JSON، مواصفات مكتوبة)
- **البحث في الويب** عن أحدث التوجهات والمكتبات والقوالب المشابهة
- **تعديل القوالب الجاهزة** بدلاً من إعادة اختراع العجلة
- **بناء أنظمة تصميم** (Design Tokens + Component Libraries)
- **تصدير نتائج بجودة إنتاجية** وليست بروتوتايب فقط

#### 6. سير العمل (Workflow)

```
TeraAgent يقرر أن المهمة تحتاج مصمم واجهات
    ↓
Tera → UIDesignerAgent
    ↓
1. مصمم يقرأ ملفات المواصفات (Design Spec / Tokens / Screenshots)
2. مصمم يبحث في الويب عن:
   - قوالب مشابهة (إن وجدت)
   - أحدث التوجهات التصميمية
   - أفضل مكتبة مناسبة للمهمة
3. مصمم يختار أو يصمم الاتجاه البصري
4. مصمم ينفذ الكود (React + Tailwind)
5. مصمم يسلم النتيجة لـ TeraAgent
    ↓
TeraAgent → DesignReviewer (ناقد) للمراجعة
    ↓
ناقد يرفع تقرير → TeraAgent يقرر القبول أو الإصلاح
```

---

#### 7. ما يمنع على هذا العميل

```
❌ لا يقرر بنفسه نطاق المشروع أو المواصفات — Tera يقرر
❌ لا يعمل بدون موافقة TeraAgent
❌ لا يتجاهل دليل التصميم المعتمد ويخترع من عنده
❌ لا يكتفي "بشيء يعمل" — يجب أن يكون جميلاً وأنيقاً
❌ لا يضيف مكتبات جديدة بدون تبرير
❌ لا يتجاهل RTL والعربية حتى لو الطلب بالإنجليزية
❌ لا ينتج "كود فقط" — ينتج تجربة بصرية كاملة
```
