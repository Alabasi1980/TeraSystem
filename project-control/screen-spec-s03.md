## 2. الجدول (Table)

| # | الاسم المعروض | الحقل في DB | النوع | قابل للفرز؟ | العرض التقديري |
|---|---|---|---|---|---|
| 1 | اسم البنك | `name` | نص | نعم | 200px |
| 2 | الفرع | `branch` | نص | نعم | 150px |
| 3 | رقم الحساب | `account_number` | نص | نعم | 150px |
| 4 | عدد الشيكات المرتبطة | *(محسوب)* | رقم | لا | 130px |

ملاحظات:
- عمود "عدد الشيكات المرتبطة" هو حقل محسوب باستخدام `COUNT` على جدول `checks` حيث `bank_id = banks.id`.
- عمود "الإجراءات" يحتوي على زر "تعديل" وزر "حذف".
- رؤوس الأعمدة بلون خلفية رمادي فاتح، مع خط واضح.
- الصفوف بارتفاع مريح مع حدود خفيفة و hover بسيط.

## 3. نموذج الإضافة/التعديل (Add/Edit Form)

يُفتح في **Modal** عند الضغط على "إضافة بنك جديد" أو "تعديل". نفس النموذج يُستخدم للحالتين.

| # | الاسم المعروض | الحقل في DB | النوع في DB | نوع HTML | Required? | Validation | القيمة الافتراضية | Placeholder |
|---|---|---|---|---|---|---|---|---|
| 1 | اسم البنك | `name` | string(100) | text | ✅ نعم | مطلوب، حد أقصى 100 حرف | — | أدخل اسم البنك |
| 2 | الفرع | `branch` | string(100) | text | ❌ لا | حد أقصى 100 حرف | — | أدخل الفرع (اختياري) |
| 3 | رقم الحساب | `account_number` | string(50) | text | ❌ لا | حد أقصى 50 حرف | — | أدخل رقم الحساب (اختياري) |
| 4 | ملاحظات | `notes` | text | textarea | ❌ لا | حد أقصى 500 حرف | — | ملاحظات (اختياري) |

قواعد النموذج:
- الحقول مرتبة حسب الترتيب أعلاه.
- كل حقل له Label واضح بالعربية.
- الحقول المطلوبة مميزة بعلامة (مثل `*` أو نجمة حمراء).
- رسائل الخطأ تظهر أسفل الحقل عند التحقق (Validation).
- Placeholder يُستخدم كتلميح إضافي (وليس بديلاً عن Label).
- زرين أسفل النموذج: **حفظ** (Primary) و**إلغاء** (Secondary).

## 4. الأزرار والإجراءات

### 4.1 زر "إضافة بنك جديد"

| البند | القيمة |
|---|---|
| النص المعروض | إضافة بنك جديد |
| الموقع | أعلى يسار الجدول (في الشريط العلوي للصفحة) |
| متى يظهر | دائمًا (ما دام المستخدم Admin) |
| السلوك | يفتح Modal فارغًا للإضافة — العنوان: "إضافة بنك جديد" |
| الإجراء في DB | INSERT INTO banks |

### 4.2 زر "تعديل"

| البند | القيمة |
|---|---|
| النص المعروض | تعديل |
| الموقع | داخل صف الجدول — عمود الإجراءات |
| متى يظهر | دائمًا — لكل صف |
| السلوك | يفتح Modal معبأً بالبيانات الحالية — العنوان: "تعديل البنك: [اسم البنك]" |
| الإجراء في DB | UPDATE banks SET ... WHERE id = ? |

### 4.3 زر "حذف"

| البند | القيمة |
|---|---|
| النص المعروض | حذف |
| الموقع | داخل صف الجدول — عمود الإجراءات |
| متى يظهر | دائمًا — لكل صف |
| السلوك | يظهر تأكيد حذف → إذا وافق المستخدم → يحذف |
| الإجراء في DB | DELETE FROM banks WHERE id = ? |
| النوع البصري | Danger — أحمر |

## 5. حالات خاصة

### 5.1 لا توجد بنوك مسجلة بعد

- يعرض رسالة: "لا توجد بنوك مسجلة بعد."
- مع زر "إضافة بنك جديد" أسفل الرسالة.

### 5.2 محاولة حذف بنك مرتبط بشيكات

- يمنع الحذف.
- يعرض رسالة: "لا يمكن حذف هذا البنك لأنه مرتبط بشيكات موجودة. قم بحذف أو إعادة ربط الشيكات المرتبطة أولاً."
- لا ينفذ DELETE.

### 5.3 نجاح العملية

| العملية | الرسالة |
|---|---|
| إضافة | تم إضافة البنك بنجاح |
| تعديل | تم تعديل البنك بنجاح |
| حذف | تم حذف البنك بنجاح |

- تظهر الرسالة لمدة 3-5 ثوانٍ (Toast / Snackbar) بلون أخضر.

### 5.4 فشل العملية

| الحالة | الرسالة |
|---|---|
| خطأ في الإدخال | يرجى التحقق من الحقول المطلوبة |
| خطأ في الخادم | حدث خطأ أثناء حفظ البيانات. حاول مرة أخرى. |
| تعارض بيانات | بنك بنفس الاسم موجود مسبقًا (إذا تم تطبيق unique على name) |

- تظهر الرسالة بلون أحمر.
- في حالة خطأ الخادم، يبقى النموذج مفتوحًا ولا يُغلق حتى لا يفقد المستخدم البيانات المدخلة.

### 5.5 تأكيد الحذف

| العنصر | النص |
|---|---|
| العنوان | تأكيد الحذف |
| نص التأكيد | هل أنت متأكد من حذف البنك "[اسم البنك]"؟ |
| زر التأكيد | حذف (Danger) |
| زر الإلغاء | إلغاء (Secondary) |

## 6. ملاحظات تنفيذية

- **الصلاحية**: Admin فقط. يضمنها Middleware الموجود في التطبيق. إذا كان `role !== 'ADMIN'`، يعيد التوجيه إلى `/checks`.
- **ترتيب البنوك**: أبجدي تصاعدي حسب `name` (ORDER BY name ASC).
- **عدد الشيكات المرتبطة**: استعلام COUNT جانبي مع GROUP BY أو حقل محسوب داخل نفس الاستعلام (LEFT JOIN مع COUNT).
- **النموذج الموحد**: Modal واحد يُستخدم للإضافة والتعديل؛ عند التعديل تُعبأ الحقول بقيم السجل الحالي، وعند الإضافة تكون خالية.
- **إعادة تحميل الجدول**: بعد أي عملية (إضافة/تعديل/حذف) يتم إعادة تحميل قائمة البنوك تلقائيًا بدون تحديث كامل للصفحة (AJAX / API call).
- **التحقق من الشيكات المرتبطة قبل الحذف**: استعلام مسبق: SELECT COUNT(*) FROM checks WHERE bank_id = ?. إذا > 0، يمنع الحذف وتظهر رسالة المنع.
- **الألوان المطابقة لـ UI/UX Guidelines**: أزرار الحذف Danger (أحمر هادئ)، أزرار الحفظ Primary (أزرق إداري)، الخلفية بيضاء، النصوص رمادية داكنة.
