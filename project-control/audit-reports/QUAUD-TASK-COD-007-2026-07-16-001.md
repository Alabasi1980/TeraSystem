## Finding F-004 Ś CAUTION: Shared IsRunning Status Not Safe Under Concurrent Operations

| Field | Value |
|---|---|
| **Finding ID** | F-004 |
| **Rule ID** | QG-COR-001 (Concurrency Correctness) |
| **Domain** | Concurrency / Thread Safety |
| **Severity** | CAUTION |
| **Location** | SyncEngineService.cs Ś SetStatus(running: true/false, ...) calls in both RunSyncOnceAsync (lines 123, 195, 220, 227) and RunSelectedMappingsAsync (lines 247, 259, 396¢400) |
| **Evidence** | Direct file read. Both methods call SetStatus(running: true, ...) on entry and SetStatus(running: false, ...) on exit. RunSelectedMappingsAsync is explicitly designed to NOT acquire the semaphore (documented in XML doc, line 237¢238), so it can run concurrently with RunSyncOnceAsync. The _statusLock protects individual field read/write atomicity, but does not protect the logical "is anything running" state. If RunSyncOnceAsync finishes while RunSelectedMappingsAsync is still executing, IsRunning is set to alse prematurely. The API consumer would momentarily see IsRunning=false while a sync is still in progress. |
| **Expected Standard** | Thread-safe status should accurately reflect whether any sync operation is running. |
| **Observed Condition** | Boolean _isRunning does not account for concurrent operations. The last method to finish sets the flag, potentially clearing it while the other is still running. |
| **Impact** | The status API could momentarily report IsRunning=false while a sync is still in progress. Self-correcting when the remaining operation finishes, but could mislead dashboard consumers during the window. |
| **Recommended Action** | Replace the boolean _isRunning with an atomic counter (e.g., Interlocked.Increment on start, Interlocked.Decrement on finish) so IsRunning returns 	rue whenever any operation is active. Alternatively, accept this as a documented design tradeoff if concurrent auto+manual sync is unlikely in practice. |
| **Changed Code / Baseline** | Existing pattern from TASK-COD-007, exposed/activated by the new RunSelectedMappingsAsync bypassing the semaphore |
| **Confidence** | High |
