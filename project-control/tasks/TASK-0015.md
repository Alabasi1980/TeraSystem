## 4. S05 Spec (from 07_SCREENS_AND_UI_STRUCTURE.md)

- Route: `/users`
- Admin only (requireAdmin on ALL actions)
- Table: اسم المستخدم, الاسم المعروض, الدور, نشط/غير نشط
- Actions: إضافة, تعديل, تفعيل/تعطيل
- No delete
- Form: username, password (on add/reset), display name, role (ADMIN/USER), active toggle

## 5. User Model (from schema.prisma)

```prisma
model User {
  id           Int       @id @default(autoincrement())
  username     String    @unique
  passwordHash String    @map("password_hash")
  displayName  String    @map("display_name")
  role         UserRole
  isActive     Boolean   @default(true) @map("is_active")
  createdAt    DateTime  @default(now()) @map("created_at")
  updatedAt    DateTime  @updatedAt @map("updated_at")
  checks       Check[]
  @@map("users")
}
```

## 6. Scope / Out of Scope

- Server Actions: listUsers, createUser, updateUser, toggleUserActive
- Users page: table, add/edit modal, active/inactive toggle
- Password hashing for new users and password resets
- Nav activation for users card

### Out of Scope
- Delete user (not in spec)
- Self-registration
- Password change by user themselves
- Email notifications
- Role management beyond simple dropdown

## 8. Sub-Task 1: Backend — Users Server Actions

### Agent
EngineeringAgent

### Allowed Write Targets
- `checks-management/app/users/actions.ts` — **CREATE** (new file)

### Required Actions

#### Types
```typescript
export type UserItem = {
  id: number
  username: string
  displayName: string
  role: string
  isActive: boolean
  checkCount: number
  createdAt: Date
}

export type UserFormData = {
  username: string
  password?: string        // required for create, optional for edit (leave empty to keep)
  displayName: string
  role: 'ADMIN' | 'USER'
  isActive?: boolean      // default true for create
}
```

#### 1. `listUsers()`
- `requireAdmin()`
- `findMany({ orderBy: { username: 'asc' }, include: { _count: { select: { checks: true } } } })`
- Map to `UserItem[]`

#### 2. `createUser(data: UserFormData)`
- `requireAdmin()`
- Validation: username required/unique/max 50, password required on create/min 6 chars, displayName required/max 100
- Hash password using `bcrypt` (import from existing `lib/auth.ts` or install)
- Create user with `isActive: true` default
- `revalidatePath('/users')`

#### 3. `updateUser(id: number, data: UserFormData)`
- `requireAdmin()`
- Validation: username required/unique (exclude self)/max 50, displayName required/max 100
- If `data.password` is provided and non-empty: validate min 6 chars and re-hash
- If `data.isActive` is provided: update it
- **Protect self**: if updating own account, don't allow deactivating self or changing own role from ADMIN
- `revalidatePath('/users')`

#### 4. `toggleUserActive(id: number)`
- `requireAdmin()`
- Fetch user, toggle `isActive`
- **Protect self**: don't allow deactivating self
- `revalidatePath('/users')`

#### 5. `getUserById(id: number)`
- `requireAdmin()`
- Return full user details

### Password Hashing
Check if `bcrypt` (or `bcryptjs`) is already installed (it was used in TASK-0004 for auth). Import it:
```typescript
import bcrypt from 'bcryptjs' // or 'bcrypt'
const salt = await bcrypt.genSalt(10)
const hash = await bcrypt.hash(password, salt)
```
