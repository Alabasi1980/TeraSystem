## Objective

إنشاء مشروع Next.js + TypeScript عامل محليًا، وإعداد Prisma للاتصال بـ PostgreSQL، وإنشاء ملف `.env.example` فقط (بدون `.env` فعلي وبدون أي نموذج بيانات).

هذه المهمة هي **أصغر وحدة تنفيذية آمنة** — تقوم فقط بتجهيز البيئة التقنية ولا تحتوي على أي Schema أو UI أو Auth.

## الأهداف المحددة

1. Scaffold مشروع Next.js + TypeScript في مجلد `checks-management` داخل الـ workspace.
2. تثبيت حزمة `prisma` و `@prisma/client`.
3. إنشاء ملف `prisma/schema.prisma` أساسي (بدون أي `model`) يحتوي فقط على:
   - `generator client` مع `provider = "prisma-client-js"`
   - `datasource db` مع `provider = "postgresql"` و `url = env("DATABASE_URL")`
4. إنشاء ملف `.env.example` يحتوي فقط على المتغير `DATABASE_URL` مع قيمة placeholder آمنة:
   ```
   DATABASE_URL="postgresql://user:password@localhost:5432/checks_management"
   ```
5. التأكد أن المشروع يعمل بأمر `npm run dev` (أو `yarn dev` / `pnpm dev` حسب ما يختاره الأمر).
