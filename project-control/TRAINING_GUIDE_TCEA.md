## <a name="2"></a>2. نظرة عامة على الآلية

### القاعدة الذهبية

السعر النهائي = تقييم موضوعي (12 عاملاً) × مضاعفات المنصة واللغة + الإضافات + هامش مخاطر

**لا تكتب سعراً من رأسك.** الآلية تحسب السعر وأنت تطبق فقط.

### المكونات الأساسية

```
Complexity Index (0-100%) ← Weighted Score / 92.5
     ↓
Base Price ← Interpolation على 7 Anchor Points
     ↓
× Platform Multiplier (Web / Mobile / PWA)
     ↓
× Language Multiplier (عربي / إنجليزي / متعدد)
     ↓
+ External Add-ons (Payment / SMS / ERP …)
     ↓
× (1 + Risk Margin)    (5% – 20%)
     ↓
× (1 + Rush Premium)   (0% – 35%)
     ↓
Final Scorecard Price
     ↓
max(Final, 100 JOD, Estimated Hours × 4 JOD) = السعر المعتمد
```

## <a name="3"></a>3. خطوات التسعير — دليل خطوة بخطوة

### الخطوة 1: تأكد من توفر المعلومات

قبل أن تبدأ، يجب أن يكون عندك:

- [ ] نطاق المشروع (Scope)
- [ ] عدد الشاشات التقريبي
- [ ] العمليات الأساسية
- [ ] المستخدمون والصلاحيات
- [ ] التقارير
- [ ] التكاملات
- [ ] النشر والاستضافة
- [ ] المطلوب بعد التسليم
- [ ] المدة الزمنية
- [ ] حدود الدعم والتعديلات

**إذا نقصت أي معلومة → لا تسعر.** اعرض تحليلاً مدفوعاً بدلاً من السعر المقطوع.

### الخطوة 2: افتح حاسبة Excel

افتح `TeraPricingCalculator.xlsx`. ستجد نفسك في صفحة **تسعير التطبيقات**.

### الخطوة 3: املأ معلومات المشروع

| الحقل | كيف تملؤه |
|-------|-----------|
| اسم المشروع | اسم المشروع كما سيظهر في العرض |
| اسم العميل | الاسم الرسمي للعميل |
| التاريخ | تاريخ إعداد العرض |
| أعدها | اسمك |

### الخطوة 4: قيّم العوامل 12

استخدم جدول **Rubric التقييم** في السياسة (Section 5) لتحديد درجة كل عامل من 0 إلى 5.

دليل سريع للعوامل:

| العامل | درجة 1-2 | درجة 3-4 | درجة 5 |
|--------|---------|---------|-------|
| عدد الشاشات | 1-7 شاشات (1-2) | 8-25 شاشة (3-4) | >25 (5) |
| منطق الأعمال | CRUD بسيط (1-2) | Workflow وموافقات (3-4) | ERP/مالي (5) |
| قاعدة البيانات | <8 جداول (1-2) | 9-30 جدولاً (3-4) | >30 أو ERP (5) |
| الصلاحيات | 1-2 دور (1-2) | 3-6 أدوار (3-4) | ديناميكية (5) |
| التقارير | 1-5 تقارير بسيطة (1-2) | تقارير + Dashboard (3-4) | BI/OLAP (5) |
| التكاملات | Email/API واحد (1-2) | تكاملان+ (3-4) | ERP/حساس (5) |
| التصميم | قالب/عادي (1-2) | مخصص (3-4) | Figma كامل (5) |
| الطباعة | PDF بسيط (1-2) | قوالب رسمية (3-4) | فواتير/عقود (5) |
| الفروع | مستخدم واحد (1-2) | أقسام/فروع (3-4) | هيكل مؤسسي (5) |
| النشر | محلي/بسيط (1-2) | Domain+SSL (3-4) | Production (5) |
| التدريب | شرح/جلسة (1-2) | تدريب فريق (3-4) | توثيق كامل (5) |
| الاختبار | يدوي بسيط (1-2) | يدوي شامل (3-4) | حمل/أمان (5) |

**هام:** أي عامل بدرجة 5 يتطلب مراجعة إلزامية قبل اعتماد السعر.

### الخطوة 5: اختر المضاعفات

من القوائم المنسدلة في Excel، اختر:

1. **المنصة** — اختر الأنسب
   - **Web Responsive جيد** ← ×1.00 (الافتراضي)
   - **Web فقط بدون Responsive** ← ×0.90
   - **Web + PWA** ← ×1.10 إلى ×1.20
   - **هواتف** ← ×1.35 إلى ×2.10

2. **اللغة** — لغة واحدة ← ×1.00 (الافتراضي)
   - عربي/إنجليزي كامل ← ×1.25

3. **هامش المخاطر** — اختر حسب وضوح النطاق
   - نطاق واضح ومكتوب ← 5%
   - تفاصيل ناقصة ← 10%
   - عميل متردد ← 15%

4. **بدل الاستعجال** — 0% إذا المدة طبيعية

### الخطوة 6: أضف الإضافات (إن وجدت)

اختر مستوى لكل Add-on من القائمة المنسدلة:

| المستوى | المعنى |
|---------|--------|
| لا شيء | غير مطلوب |
| بسيط | مزود موثق، حالة استخدام واحدة |
| متوسط | أكثر من سيناريو، إعدادات متعددة |
| معقد / حساس | API غير موثق، بيانات حساسة |

**قاعدة التناسب الإلزامية:** مجموع Add-ons يجب ألا يتجاوز 50% من Base Price.
- استثناء: إذا كان Add-on واحد معقداً.
- استثناء: إذا المشروع أقل من 250 JOD.

### الخطوة 7: أدخل الساعات المقدرة

قدّر ساعات التطوير الإجمالية (تصميم + برمجة + اختبار + نشر + تدريب + إدارة).

السعر المعتمد لا يمكن أن يقل عن: `الساعات × 4 JOD`

تأكد من أن تقديرك واقعي وليس متفائلاً جداً.

### الخطوة 8: اقرأ النتيجة

Excel سيحسب لك:

- **السعر المعتمد النهائي (JOD)** ← هذا هو السعر الذي تعمل به
- **فحص التناسب** ← سيعلمك إذا تجاوزت الـ 50%
- **تصنيف السعر** ← يحدد خطة الدفع المناسبة

### الخطوة 9: استخدم قائمة الاعتماد

قبل إرسال السعر للعميل، تأكد من الإجابة بـ "نعم" على جميع أسئلة القائمة في السياسة (Section 29 — 14 سؤالاً).

### الخطوة 10: املأ قالب الخطاب الرسمي

افتح `letterhead-master-fixed-print.html` واملأ:
- معلومات المشروع
- نطاق العمل (باختصار)
- تفاصيل السعر (انقلها من Excel)
- الشروط (موجودة مسبقاً)
- التوقيع

## <a name="6"></a>6. المثال التطبيقي: مشروع الموثوق (Mawthooq)

### البيانات

| المعلومة | القيمة |
|---------|:------:|
| نوع المشروع | نظام إدارة أعمال الصيانة |
| الشاشات | 8-15 شاشة |
| المستخدمون | 4 أدوار، ~7 مستخدمين |
| العمليات | Workflow لطلبات الصيانة (10 حالات) |
| التقارير | 3 تقارير أساسية |
| التكاملات | لا توجد |
| التصميم | مخصص بسيط |
| PDF | إيصال بسيط |
| النشر | سحابي + Domain |

### تقييم العوامل

شاهد `PRICING_SCORECARD_APPLICATION_MAWTHOOQ.md` للحصول على التفاصيل الكاملة.

النتيجة النهائية: **~500 JOD** (كانت سابقاً 1,605 JOD قبل تعديل السياسة).

### ما تعلمناه من هذا المثال

1. **الدرجات تعتمد على Rubric محدد** — لا تخمن، ارجع للجدول.
2. **Base Price يتغير خطياً** — كلما زاد Index، زاد السعر لكن ليس بشكل عشوائي.
3. **Platform multiplier ×1.00** للـ Web Responsive هو الأساس الآن.
4. **الـ Min Profitable Price** هو شبكة الأمان — لا تنزل تحته.

## <a name="7"></a>7. الأخطاء الشائعة وتجنبها

### الخطأ 1: التقييم العاطفي
❌ "هذا العميل صديق، أعطيه درجة منخفضة"
✅ قيم حسب Rubric فقط، بغض النظر عن العلاقة

### الخطأ 2: تجاهل المضاعفات
❌ نسيت تغيير مضاعف المنصة من Web إلى Mobile
✅ اختر من القائمة المنسدلة في Excel لكل مشروع

### الخطأ 3: الاحتساب المزدوج
❌ حسبت تكامل SMS في Scorecard + أضفته كـ Add-on
✅ السياسة تمنع هذا (Section 13). اختر أحدهما.

### الخطأ 4: تجاهل Minimum Profitable Price
❌ السعر 400 JOD لكن الساعات = 120 ساعة ← 120×4=480 JOD (أعلى!)
✅ تأكد دائماً من أن Approved Price ≥ 480

### الخطأ 5: إضافة Add-on كبير لمشروع صغير
❌ مشروع بـ 200 JOD + Payment Gateway بـ 100 JOD = 50%
✅ إذا تجاوز 50% من Base Price، طبق قاعدة التناسب أو استثنِ بتوثيق

### الخطأ 6: نسيان توقيع العميل على النطاق
❌ السعر صحيح لكن النطاق غير موقع → تغييرات بلا نهاية
✅ لا تبدأ التنفيذ قبل توقيع النطاق + الدفعة الأولى

## <a name="8"></a>8. قائمة الاعتماد النهائية

**قبل إرسال أي عرض سعر، أجب على هذه الأسئلة:**

| # | السؤال |
|:-:|--------|
| 1 | هل النطاق مكتوب وواضح؟ |
| 2 | هل تم تقييم العوامل 12؟ |
| 3 | هل تم منع الاحتساب المزدوج؟ |
| 4 | هل تم تحديد Platform Multiplier؟ |
| 5 | هل تم تحديد Language Multiplier؟ |
| 6 | هل تم توثيق Add-ons؟ |
| 6.1 | هل تم التحقق من قاعدة التناسب (Add-ons ≤ 50% من Base Price)؟ |
| 7 | هل تم تحديد Risk Margin؟ |
| 8 | هل يوجد استعجال وتم احتساب Rush Premium؟ |
| 9 | هل تم حساب Minimum Profitable Price؟ |
| 10 | هل السعر لا يقل عن 100 JOD؟ |
| 11 | هل شروط الدفع واضحة؟ |

## <a name="9"></a>9. أسئلة متوقعة وإجاباتها

### س: كم سعر تطبيق بسيط بـ 5 شاشات و CRUD فقط؟
أدخل درجات: معظمها 1-2. Complexity Index ≈ 10-15%. Base Price ≈ 100-150 JOD. السعر النهائي ≈ 100-150 JOD.

### س: العميل يريد تخفيض 20%؟
التخفيض الأقصى المسموح: 15% وفقط لسبب تجاري واضح. أكثر من 15% = تقليل النطاق أو مراحل.

### س: مشروع فوق 80% Index؟
خارج الآلية. يحتاج SOW أو دراسة تفصيلية.

### س: عميل يريد السعر دون إعطائي تفاصيل؟
لا تسعر. اعرض تحليلاً مدفوعاً (35-75 JOD للصغير، 100-200 للمتوسط).

### س: هل Add-on Payment Gateway دائماً بقيمة 100 JOD؟
إذا كان بسيطاً (مزود موثق، حالة استخدام واحدة) ← 100 JOD.
إذا كان معقداً (API غير موثق، بيانات حساسة) ← 350+ JOD.

### س: السعر النهائي أقل من 100 JOD؟
الحد الأدنى هو 100 JOD. ارفع السعر إلى 100 JOD.

### س: العميل يحتاج دعم شهري بعد التسليم؟
الدعم منفصل عن سعر التطوير. ابدأ من 20-50 JOD/شهر للمشاريع الصغيرة.

### س: طلب إضافة شاشة بعد توقيع العقد؟
هذا Change Request. حسب حجمه: بالساعة (6 JOD/hr) أو بعرض مصغر.

## <a name="10"></a>10. قرارات سريعة — متى تخرج عن الآلية

| الحالة | القرار |
|--------|--------|
| Complexity Index > 80% | SOW / دراسة تفصيلية |
| نطاق غير مستقر | تحليل مدفوع أو Time & Material |
| العميل لا يعرف ماذا يريد | تحليل مدفوع |
| تكاملات حساسة غير موثقة | Discovery / SOW |
| Native Mobile منفصل بالكامل | تسعير مستقل |
| ضغط زمني غير واقعي | رفض العرض أو SOW خاص |
| عامل بدرجة 5 (Log الأعمال، DB، صلاحيات، تكاملات) | مراجعة إلزامية — قد يتحول لتحليل مدفوع |

## ملخص سريع — قبل كل مشروع

```
1. تأكد من وجود نطاق واضح
2. افتح Excel → املأ الخلايا الصفراء ← اقرأ النتيجة
3. طبق قائمة الاعتماد (14 سؤالاً)
4. املأ قالب الخطاب الرسمي: letterhead-master-fixed-print.html
6. أرسل للعميل عبر Majed
7. لا تبدأ قبل التوقيع + الدفعة الأولى
```

**تذكير إلزامي:** أي مستند ترسله للزبون — عرض سعر، عقد، خطاب، أي شيء — يجب أن يكون على قالب الخطاب الرسمي للمؤسسة. هذا ليس خياراً.
