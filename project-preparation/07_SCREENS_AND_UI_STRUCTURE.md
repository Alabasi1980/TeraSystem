# 07_SCREENS_AND_UI_STRUCTURE.md

## هيكل الشاشات وواجهة المستخدم

---

### 1. قائمة الشاشات بعد تقليل التضخيم

تم تقليل الشاشات من 17 شاشة إلى **5 شاشات فقط** مناسبة لـ MVP. الإضافة والتعديل والتفاصيل وتغيير الحالة تُنفذ داخل نفس شاشة الإدارة باستخدام نموذج جانبي أو نافذة منبثقة، ولا تحتاج شاشات مستقلة.

| المعرف | اسم الشاشة | الوظيفة |
|---|---|---|
| S01 | تسجيل الدخول | دخول المستخدم للنظام |
| S02 | إدارة الشيكات | قائمة الشيكات + الفلاتر + الإضافة/التعديل + التفاصيل + تغيير الحالة + الطباعة |
| S03 | إدارة البنوك | قائمة البنوك + إضافة/تعديل/حذف بنك من نفس الشاشة |
| S04 | إدارة الجهات | قائمة الجهات + إضافة/تعديل/حذف جهة من نفس الشاشة |
| S05 | إدارة المستخدمين | قائمة المستخدمين + إضافة/تعديل/تعطيل مستخدم — Admin فقط |

---

### 2. خريطة التنقل

```
[تسجيل الدخول S01]
       |
[إدارة الشيكات S02]  ← الشاشة الرئيسية بعد الدخول
       |
       ├── [إدارة البنوك S03]        ← Admin فقط
       ├── [إدارة الجهات S04]        ← Admin فقط
       └── [إدارة المستخدمين S05]    ← Admin فقط
```

### 3. تفاصيل كل شاشة

#### S01 — تسجيل الدخول (Login)

| العنصر | التفاصيل |
|---|---|
| الوظيفة | دخول المستخدم للنظام |
| الحقول | اسم المستخدم، كلمة المرور |
| الأزرار | دخول |
| ملاحظات | لا يوجد تسجيل عام — Admin فقط يضيف المستخدمين |
| الصلاحية | الجميع |

#### S02 — إدارة الشيكات (Checks Management)

| العنصر | التفاصيل |
|---|---|
| الوظيفة | الشاشة الرئيسية للنظام: عرض، بحث، فلترة، إضافة، تعديل، تفاصيل، تغيير حالة، وطباعة كشف الشيكات |
| ملخص أعلى الشاشة | عدد الشيكات، إجمالي المبالغ، عدد الشيكات المستحقة قريبًا، عدد الشيكات المرتجعة |
| الحقول في الجدول | رقم الشيك، النوع (صادر/وارد)، البنك، الجهة، المبلغ، تاريخ الاستحقاق، الحالة، تاريخ الإضافة |
| الأزرار | إضافة شيك، تعديل، تفاصيل، تغيير حالة، حذف (Admin فقط)، طباعة الكشف |
| الفلاتر | حسب النوع، الحالة، البنك، الجهة، من تاريخ - إلى تاريخ (تاريخ الاستحقاق) |
| الفرز | ترتيب تنازلي حسب تاريخ الإضافة (الأحدث أولاً) |
| نموذج الإضافة/التعديل | النوع، البنك، الجهة، رقم الشيك، المبلغ، تاريخ الإصدار، تاريخ الاستحقاق، ملاحظات |
| التفاصيل | تظهر في نفس الشاشة عبر نافذة/لوحة جانبية ولا تحتاج صفحة مستقلة |
| تغيير الحالة | يظهر كنافذة صغيرة من داخل صف الشيك أو تفاصيله |
| الطباعة | طباعة القائمة الحالية بعد الفلاتر عبر أمر طباعة المتصفح، ولا تحتاج شاشة مستقلة |
| التحقق (Validation) | جميع الحقول الأساسية مطلوبة، المبلغ > 0، تاريخ الاستحقاق >= تاريخ الإصدار |
| الصلاحية | الجميع |

#### S03 — إدارة البنوك (Banks Management)

| العنصر | التفاصيل |
|---|---|
| الوظيفة | إدارة البنوك من شاشة واحدة |
| المحتوى | جدول: اسم البنك، الفرع، رقم الحساب، عدد الشيكات المرتبطة |
| نموذج الإضافة/التعديل | اسم البنك (مطلوب)، الفرع (اختياري)، رقم الحساب (اختياري)، ملاحظات |
| الأزرار | إضافة، تعديل، حذف |
| القيود | يمنع حذف بنك مرتبط بشيكات |
| الصلاحية | Admin فقط |

#### S04 — إدارة الجهات (Parties Management)

| العنصر | التفاصيل |
|---|---|
| الوظيفة | إدارة الجهات من شاشة واحدة |
| المحتوى | جدول: اسم الجهة، الهاتف، عدد الشيكات المرتبطة |
| نموذج الإضافة/التعديل | اسم الجهة (مطلوب)، الهاتف (اختياري)، ملاحظات |
| الأزرار | إضافة، تعديل، حذف |
| القيود | يمنع حذف جهة مرتبطة بشيكات |
| الصلاحية | Admin فقط |

#### S05 — إدارة المستخدمين (Users Management) — Admin فقط

| العنصر | التفاصيل |
|---|---|
| المحتوى | جدول: اسم المستخدم، الاسم المعروض، الدور، نشط/غير نشط |
| نموذج الإضافة/التعديل | اسم المستخدم، كلمة المرور عند الإضافة/إعادة التعيين، الاسم المعروض، الدور، الحالة |
| الأزرار | إضافة، تعديل، تعطيل/تفعيل |
| الصلاحية | Admin فقط |

### 4. حالات خاصة

| الحالة | السلوك |
|---|---|
| **لا توجد شيكات** | رسالة "لا توجد شيكات مسجلة بعد" + زر "إضافة شيك جديد" |
| **لا توجد بنوك** | عند إضافة شيك — إذا لم يتم إضافة أي بنك بعد، يعرض رسالة "يرجى إضافة بنك أولاً" |
| **لا توجد جهات** | نفس سلوك البنوك — يطلب إضافة جهة أولاً |
| **شيك في حالة نهائية** | إخفاء أو تعطيل أزرار التعديل وتغيير الحالة |
| **محاولة حذف بنك أو جهة مرتبطة بشيكات** | رسالة تمنع الحذف: "لا يمكن حذف هذا العنصر لأنه مرتبط بشيكات موجودة" |
| **جلسة منتهية** | إعادة التوجيه إلى شاشة تسجيل الدخول |

### 5. ملاحظات تجربة المستخدم

- **القوائم:** شريط جانبي أو علوي يحتوي على: الشيكات، البنوك، الجهات، المستخدمين. شاشة الشيكات هي الشاشة الرئيسية ولا توجد Dashboard مستقلة في MVP.
- **اللغة:** واجهة باللغة العربية (RTL).
- **الألوان:** تصميم إداري بسيط ونظيف — يفضل ألوان هادئة (أزرق/رمادي/أبيض).
- **البحث السريع:** حقل بحث في قائمة الشيكات يبحث في رقم الشيك أو اسم الجهة.
- **حذف العناصر:** طلب تأكيد قبل الحذف (تأكيد: "هل أنت متأكد من حذف هذا العنصر؟").
- **تحديث الحالة:** يفضل أن يكون سريعًا (نقرة واحدة بعد اختيار الحالة الجديدة).
- **رسائل النجاح/الفشل:** بعد كل عملية (إضافة، تعديل، حذف) تظهر رسالة نجاح أو خطأ واضحة.
