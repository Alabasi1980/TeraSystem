# BLUEPRINT_OPEN_QUESTIONS — نظام إدارة طلبات الصيانة الداخلي

> الأسئلة المفتوحة التي تحتاج إجابة قبل أو أثناء مرحلة التحضير الرسمي.
>
> **ملاحظة:** هذه الأسئلة مبنية على تحليل حزمة التسليم (TERA_HANDOFF_PACKAGE) والملفات المرتبطة بها.

---

## الأسئلة المعلقة

### 1. ما بيئة الاستضافة النهائية؟

| الحقل | القيمة |
|-------|--------|
| **السؤال** | هل سيعمل التطبيق على سيرفر داخلي داخل الشركة، أم على استضافة سحابية (VPS)، أم على سيرفر مشترك؟ |
| **الأثر** | يؤثر على اختيار قاعدة البيانات، إعدادات النشر، والأمان |
| **الافتراض الحالي** | استضافة محلية (On-Premise / Local Network) |
| **لمن السؤال** | العميل (عبر Majed) |
| **الأولوية** | 🟡 متوسطة |

### 2. ما قائمة الأقسام/المواقع المعتمدة؟

| الحقل | القيمة |
|-------|--------|
| **السؤال** | ما هي الأقسام أو المواقع الداخلية التي سيعمل بها الموظفون؟ (مثلاً: قسم الصيانة، قسم الكهرباء، المخزن، الإدارة، إلخ) |
| **الأثر** | يؤثر على حقل "القسم" في نموذج الطلب وفلترة الطلبات |
| **الافتراض الحالي** | غير محدد — سيحتاج إلى قائمة من العميل |
| **لمن السؤال** | العميل (عبر Majed) |
| **الأولوية** | 🟡 متوسطة |

### 3. هل الموظف يستطيع تعديل طلبه بعد التقديم؟

| الحقل | القيمة |
|-------|--------|
| **السؤال** | هل يُسمح للموظف بتعديل طلب الصيانة بعد إرساله وقبل أن يبدأ مسؤول الصيانة بمعالجته؟ |
| **الأثر** | يؤثر على صلاحيات الموظف وواجهة المستخدم |
| **الافتراض الحالي** | الموظف يمكنه تعديل الطلب فقط قبل المعالجة (مذكور في SCOPE_SUMMARY) |
| **لمن السؤال** | Majed |
| **الأولوية** | 🟡 متوسطة |

### 4. هل هناك سياسة محددة لكلمات المرور؟

| الحقل | القيمة |
|-------|--------|
| **السؤال** | هل هناك متطلبات محددة من العميل لكلمات المرور (طول، تعقيد، تغيير دوري) أم نترك الإعدادات الافتراضية الآمنة؟ |
| **الأثر** | يؤثر على سياسة المصادقة |
| **الافتراض الحالي** | لا سياسة محددة — إعدادات افتراضية آمنة |
| **لمن السؤال** | العميل (عبر Majed) |
| **الأولوية** | 🟢 منخفضة |

### 5. هل هناك حاجة لسجل تغييرات الحالة في MVP؟

| الحقل | القيمة |
|-------|--------|
| **السؤال** | هل نضيف جدولاً لتسجيل تاريخ تغييرات حالة الطلب (Status History) في النسخة الأولى، أم نكتفي بعرض الحالة الحالية فقط؟ |
| **الأثر** | يؤثر على حجم الـ MVP إذا أضفناه |
| **الافتراض الحالي** | اختياري — يُقرر في التحضير التفصيلي |
| **لمن السؤال** | Majed |
| **الأولوية** | 🟢 منخفضة |

### 6. ما هي الأولويات المعتمدة للطلبات؟

| الحقل | القيمة |
|-------|--------|
| **السؤال** | هل الأولويات المعتمدة هي فقط "عاجل، عادي، منخفض" أم هناك حاجة لمستويات إضافية؟ |
| **الأثر** | يؤثر على نموذج إنشاء الطلب والفلترة |
| **الافتراض الحالي** | 3 مستويات: عاجل، عادي، منخفض |
| **لمن السؤال** | العميل (عبر Majed) |
| **الأولوية** | 🟢 منخفضة |
