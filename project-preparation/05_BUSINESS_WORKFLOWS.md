# 05_BUSINESS_WORKFLOWS.md

## مسارات العمل — دورة حياة الشيك

---

### 1. أنواع الشيكات

| النوع | المعنى | مثال |
|---|---|---|
| **صادر (Issue)** | شيك صدر من المؤسسة لصالح جهة أخرى (نحن دافع) | دفع إيجار، تسديد فاتورة مورد |
| **وارد (Received)** | شيك استلمته المؤسسة من جهة أخرى (نحن مستلم) | تحصيل من عميل، استلام دفعة |

كلا النوعين يستخدمان نفس قائمة الحالات لتبسيط الـ MVP، لكن معنى بعض الحالات يختلف حسب نوع الشيك. هذا يمنع إنشاء مسارين منفصلين ويُبقي التطبيق بسيطًا.

---

### 2. حالات الشيك (Statuses)

| المعرف | الحالة | المعنى |
|---|---|---|
| `REGISTERED` | مسجل | تم إدخال الشيك في النظام — الحالة الابتدائية |
| `IN_HAND` | مستلم/مسلّم | للوارد: تم استلام الشيك فعليًا. للصادر: تم تسليم الشيك للمستفيد |
| `CASHED` | مصرف | للوارد: تم تحصيل قيمة الشيك. للصادر: تم خصم قيمة الشيك من الحساب |
| `RETURNED` | مرتجع | للوارد: رجع الشيك من البنك/لم يتم تحصيله. للصادر: أعيد الشيك إلينا أو لم يُصرف |
| `CANCELLED` | ملغي | تم إلغاء الشيك قبل صرفه |

---

### 3. مسارات الانتقال بين الحالات

#### المسار الطبيعي (الأساسي)

```
[مسجل] → [مستلم/مسلّم] → [مصرف]
```

#### المسارات البديلة

```
[مسجل] → [مستلم/مسلّم] → [مرتجع] → [مصرف]  (إذا أعيد تقديمه/معالجته ثم صرف)
[مسجل] → [مستلم/مسلّم] → [مرتجع]            (إذا لم يتم صرفه)
[مسجل] → [مستلم/مسلّم] → [ملغي]              (إذا تم إلغاؤه قبل الصرف)
[مسجل] → [ملغي]                              (إذا تم إلغاؤه قبل الاستلام/التسليم)
```

---

### 4. مصفوفة الانتقالات المسموحة

| من \ إلى | مسجل | مستلم/مسلّم | مصرف | مرتجع | ملغي |
|---|---|---|---|---|---|
| **مسجل** | — | ✅ | ❌ | ❌ | ✅ |
| **مستلم/مسلّم** | ❌ | — | ✅ | ✅ | ✅ |
| **مصرف** | ❌ | ❌ | — | ❌ | ❌ |
| **مرتجع** | ❌ | ✅ (إعادة تقديم/تسليم) | ✅ (إذا تمت المعالجة والصرف) | — | ✅ |
| **ملغي** | ❌ | ❌ | ❌ | ❌ | — |

> **ملاحظة مهمة:** الحالة "مصرف" و "ملغي" هما حالتان **نهائيتان** — لا يمكن تغيير الشيك بعد وصوله إلى أي منهما.

---

### 5. من ينفذ كل خطوة

| الخطوة | المسموح بالتنفيذ |
|---|---|
| تسجيل شيك جديد | Admin + User |
| تعديل شيك (في حالة غير نهائية) | Admin + User |
| حذف شيك | Admin فقط |
| تغيير الحالة من مسجل إلى مستلم/مسلّم | Admin + User |
| تغيير الحالة من مسجل إلى ملغي | Admin + User |
| تغيير الحالة من مستلم/مسلّم إلى مصرف | Admin + User |
| تغيير الحالة من مستلم/مسلّم إلى مرتجع | Admin + User |
| تغيير الحالة من مستلم/مسلّم إلى ملغي | Admin + User |
| تغيير الحالة من مرتجع إلى مستلم/مسلّم | Admin + User |
| تغيير الحالة من مرتجع إلى مصرف | Admin + User |
| تغيير الحالة من مرتجع إلى ملغي | Admin + User |

---

### 6. شروط الانتقال

| الانتقال | الشرط |
|---|---|
| مسجل ← مستلم/مسلّم | للوارد: عند استلام الشيك فعليًا. للصادر: عند تسليم الشيك للمستفيد |
| مسجل ← ملغي | يجب ألا يكون الشيك قد استلم أو صرف |
| مستلم/مسلّم ← مصرف | للوارد: عند التحصيل. للصادر: عند الخصم من الحساب |
| مستلم/مسلّم ← مرتجع | يجب ذكر سبب الارتجاع إن توفر (اختياري في MVP) |
| مرتجع ← مستلم/مسلّم | إعادة تقديم الشيك أو إعادة تسليمه بعد معالجة السبب |

---

### 6.1 توضيح اختلاف معنى الحالات حسب نوع الشيك

| الحالة | الشيك الوارد | الشيك الصادر |
|---|---|---|
| مسجل | تم إدخال بيانات الشيك في النظام | تم إدخال بيانات الشيك في النظام |
| مستلم/مسلّم | تم استلام الشيك من العميل/الجهة | تم تسليم الشيك للمستفيد |
| مصرف | تم تحصيل قيمة الشيك | تم خصم قيمة الشيك من حسابنا |
| مرتجع | لم يتم تحصيل الشيك أو رجع من البنك | أعيد الشيك إلينا أو لم يُصرف |
| ملغي | تم إلغاء التعامل مع الشيك قبل التحصيل | تم إلغاء الشيك قبل الصرف |

---

### 7. حالات الرفض أو الإلغاء

| الحالة | ماذا يحدث؟ |
|---|---|
| شيك مصرف | لا يمكن تغيير الحالة، لا يمكن حذف الشيك، لا يمكن تعديل بياناته |
| شيك ملغي | لا يمكن تغيير الحالة، لا يمكن حذف الشيك، لا يمكن تعديل بياناته |
| شيك مرتجع | يمكن إعادة تقديمه أو إلغاؤه — لا يمكن صرفه مباشرة |

---

### 8. الاستثناءات المهمة

- إذا تم حذف بنك أو جهة مرتبطة بشيكات، يمنع النظام الحذف ويعرض رسالة توضح السبب.
- عند تغيير حالة الشيك إلى "مصرف" أو "ملغي"، يُغلق الشيك نهائيًا ولا يمكن الرجوع عن ذلك.
- يجب تسجيل تاريخ العملية عند كل تغيير حالة (يسجل النظام متى تم التغيير).
