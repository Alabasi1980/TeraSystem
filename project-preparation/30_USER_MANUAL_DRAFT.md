## 1. مقدمة عن النظام

**نظام إدارة الشيكات** هو تطبيق ويب يساعدك على تسجيل ومتابعة الشيكات الصادرة والواردة في مؤسستك.

### ما الذي يمكنك فعله؟

- تسجيل شيكات جديدة (صادرة أو واردة)
- متابعة حالة كل شيك (مسجل ← مستلم ← مصرف / مرتجع / ملغي)
- البحث والفلترة بين الشيكات
- طباعة كشف بالشيكات
- إدارة البنوك والجهات المرتبطة بالشيكات
- إدارة المستخدمين (للمدير)

## 2. تسجيل الدخول

### 2.1 شاشة الدخول

عند فتح التطبيق لأول مرة، ستظهر شاشة تسجيل الدخول:

```
┌─────────────────────────────┐
│     نظام إدارة الشيكات       │
│       تسجيل الدخول           │
│                             │
│  اسم المستخدم               │
│  [____________________]     │
│                             │
│  كلمة المرور                │
│  [____________________]     │
│                             │
│  ┌─────────────────────┐   │
│  │       دخول          │   │
│  └─────────────────────┘   │
└─────────────────────────────┘
```

### 2.2 خطوات تسجيل الدخول

1. أدخل **اسم المستخدم** (الذي زودك به مدير النظام).
2. أدخل **كلمة المرور**.
3. اضغط على زر **"دخول"**.
4. إذا كانت البيانات صحيحة، سيتم نقلك إلى الصفحة الرئيسية.

### 2.3 إذا نسيت كلمة المرور

تواصل مع مدير النظام (Admin) لإعادة تعيين كلمة المرور. لا توجد خاصية استعادة كلمة مرور ذاتية في هذه النسخة.

### 2.4 تسجيل الخروج

اضغط على رابط **"تسجيل خروج"** في أعلى يسار الصفحة الرئيسية.

## 3. الصفحة الرئيسية والتنقل

### 3.1 ماذا ترى بعد تسجيل الدخول؟

- في الأعلى: اسم المستخدم ودوره (Admin / User) وزر تسجيل خروج.
- في المنتصف: 4 بطاقات تنقل:

| البطاقة | الوصف | متاحة لـ |
|---|---|---|
| **البنوك** | إدارة البنوك المسجلة في النظام | Admin فقط |
| **الجهات** | إدارة الجهات المرتبطة بالشيكات | Admin فقط |
| **الشيكات** | إدارة الشيكات بأنواعها وحالاتها | جميع المستخدمين |
| **المستخدمين** | إدارة المستخدمين والصلاحيات | Admin فقط |

### 3.2 كيفية التنقل

اضغط على أي بطاقة للانتقال إلى الشاشة المطلوبة.

> **ملاحظة:** إذا كنت مستخدمًا عاديًا (User)، فستظهر لك بطاقات البنوك والجهات والمستخدمين ولكن لا يمكنك الدخول إليها (سيتم إعادة توجيهك للصفحة الرئيسية).

## 4. إدارة الشيكات

### 4.1 شاشة الشيكات

هذه هي الشاشة الرئيسية التي ستستخدمها يوميًا. تحتوي على:

1. **شريط العنوان** — يحتوي على عنوان "إدارة الشيكات" وزرين: "طباعة الكشف" و"إضافة شيك جديد"
2. **بطاقات الملخص** — 4 بطاقات تعرض إحصائيات سريعة
3. **شريط الفلاتر** — للبحث والتصفية
4. **جدول الشيكات** — يعرض جميع الشيكات المطابقة للفلاتر

### 4.2 بطاقات الملخص

| البطاقة | اللون | المعنى |
|---|---|---|
| عدد الشيكات | أزرق | إجمالي عدد الشيكات المسجلة |
| إجمالي المبالغ | أخضر | مجموع مبالغ جميع الشيكات |
| مستحق قريبًا | أصفر | عدد الشيكات التي تستحق خلال 7 أيام (غير المصرفة والملغية) |
| شيكات مرتجعة | أحمر | عدد الشيكات المرتجعة |

### 4.3 شريط الفلاتر

يمكنك تصفية الشيكات باستخدام:

| الفلتر | النوع | الوصف |
|---|---|---|
| النوع | قائمة منسدلة | الكل / صادر / وارد |
| الحالة | قائمة منسدلة | الكل / مسجل / مستلم / مصرف / مرتجع / ملغي |
| البنك | قائمة منسدلة | اختر بنكًا معينًا |
| الجهة | قائمة منسدلة | اختر جهة معينة |
| من تاريخ | تاريخ | بداية فترة تاريخ الاستحقاق |
| إلى تاريخ | تاريخ | نهاية فترة تاريخ الاستحقاق |
| بحث | نص | بحث برقم الشيك أو اسم الجهة |

**زر "إعادة تعيين"** يلغي جميع الفلاتر ويعرض الكل.

> ملاحظة: البحث يعمل تلقائيًا بعد 0.5 ثانية من التوقف عن الكتابة. باقي الفلاتر تعمل فورًا.

### 4.4 جدول الشيكات

يحتوي الجدول على الأعمدة التالية:
- رقم الشيك
- النوع (صادر / وارد) — مع شارة ملونة
- البنك
- الجهة
- المبلغ — مع تنسيق آلي بفاصل الألوف
- تاريخ الاستحقاق
- الحالة — مع شارة ملونة
- الإجراءات — أزرار للتعامل مع كل شيك

### 4.5 إضافة شيك جديد

1. اضغط على زر **"إضافة شيك جديد"** (أزرق في أعلى الشاشة).
2. ستظهر نافذة منبثقة تحتوي على الحقول التالية:

| الحقل | إلزامي؟ | الوصف |
|---|---|---|
| النوع | ✅ | اختر "صادر" (شيك دفعته المؤسسة) أو "وارد" (شيك استلمته المؤسسة) |
| البنك | ✅ | اختر البنك من القائمة |
| الجهة | ✅ | اختر الجهة (الطرف الآخر للشيك) |
| رقم الشيك | ✅ | أدخل رقم الشيك (حد أقصى 50 حرف) |
| المبلغ | ✅ | أدخل المبلغ (يجب أن يكون أكبر من 0) |
| تاريخ الإصدار | ✅ | تاريخ تحرير الشيك |
| تاريخ الاستحقاق | ✅ | تاريخ استحقاق الشيك (يجب أن يكون ≥ تاريخ الإصدار) |
| ملاحظات | لا | أي ملاحظات إضافية (حد أقصى 500 حرف) |

3. املأ الحقول المطلوبة واضغط **"حفظ"**.
4. بعد الحفظ الناجح، سيظهر إشعار أخضر ويتم تحديث القائمة تلقائيًا.

### 4.6 تعديل شيك

1. في جدول الشيكات، اضغط على زر **"تعديل"** في صف الشيك المطلوب.
2. ستظهر نافذة تعديل بنفس حقول الإضافة، مع البيانات الحالية.
3. عدّل البيانات المطلوبة واضغط **"حفظ"**.

> ⛔ لا يمكن تعديل الشيكات في الحالة النهائية (مصرف / ملغي).

### 4.7 تغيير حالة الشيك

1. اضغط على زر **"تغيير حالة"** في صف الشيك المطلوب.
2. ستظهر نافذة تظهر الحالة الحالية وتاريخ آخر تغيير.
3. اختر **الحالة الجديدة** من القائمة (تظهر فقط الحالات المسموح الانتقال إليها).
4. اختياريًا: أضف **ملاحظة عن الحالة**.
5. اضغط **"حفظ"**.

#### حالات الانتقال المسموحة:

| من | إلى |
|---|---|
| مسجل (REGISTERED) | مستلم/مسلّم (IN_HAND) أو ملغي (CANCELLED) |
| مستلم/مسلّم (IN_HAND) | مصرف (CASHED) أو مرتجع (RETURNED) أو ملغي (CANCELLED) |
| مرتجع (RETURNED) | مستلم/مسلّم (IN_HAND) أو مصرف (CASHED) أو ملغي (CANCELLED) |
| مصرف (CASHED) | **حالة نهائية — لا يمكن التغيير** |
| ملغي (CANCELLED) | **حالة نهائية — لا يمكن التغيير** |

### 4.8 حذف شيك (Admin فقط)

1. اضغط على زر **"حذف"** في صف الشيك المطلوب.
2. ستظهر نافذة تأكيد.
3. اضغط **"حذف"** للتأكيد.

> ⛔ لا يمكن حذف الشيكات في الحالة النهائية (مصرف / ملغي).
> زر الحذف يظهر فقط لمستخدمي ADMIN وللشيكات غير النهائية.

### 4.9 عرض تفاصيل الشيك

1. اضغط على زر **"تفاصيل"** في صف الشيك المطلوب.
2. ستظهر نافذة تعرض كامل بيانات الشيك:
   - النوع، البنك، الجهة، رقم الشيك، المبلغ
   - تاريخ الإصدار، تاريخ الاستحقاق
   - الحالة الحالية مع ملاحظة الحالة وتاريخ آخر تغيير
   - ملاحظات إضافية
   - المستخدم المُنشئ وتاريخ الإنشاء

### 4.10 طباعة كشف الشيكات

1. قم بتصفية الشيكات حسب رغبتك.
2. اضغط على زر **"طباعة الكشف"** في أعلى الشاشة.
3. ستظهر نافذة الطباعة في المتصفح.
4. اختر إعدادات الطباعة المناسبة واضغط طباعة.

> ملاحظة: عند الطباعة، تظهر الشيكات المفلترة فقط في جدول طباعة نظيف بدون أزرار.

## 5. إدارة البنوك (Admin فقط)

### 5.1 شاشة البنوك

تتيح لك إدارة قائمة البنوك التي يتم إصدار أو استلام الشيكات من خلالها.

### 5.2 إضافة بنك جديد

1. اضغط على **"إضافة بنك جديد"**.
2. أدخل:
   - **اسم البنك** (إلزامي — حد أقصى 100 حرف)
   - **الفرع** (اختياري)
   - **رقم الحساب** (اختياري)
   - **ملاحظات** (اختياري — حد أقصى 500 حرف)
3. اضغط **"حفظ"**.

### 5.3 تعديل بنك

اضغط على **"تعديل"** في صف البنك المطلوب، عدّل البيانات، ثم اضغط حفظ.

### 5.4 حذف بنك

اضغط على **"حذف"** — لا يمكن حذف بنك مرتبط بشيكات موجودة. سترى رسالة توضح ذلك.

### 5.5 أعمدة الجدول

| العمود | الوصف |
|---|---|
| اسم البنك | اسم البنك |
| الفرع | الفرع (إن وجد) |
| رقم الحساب | رقم حساب البنك (إن وجد) |
| الشيكات المرتبطة | عدد الشيكات المرتبطة بهذا البنك |
| الإجراءات | أزرار تعديل / حذف |

## 6. إدارة الجهات (Admin فقط)

### 6.1 شاشة الجهات

تتيح لك إدارة قائمة الجهات (الأشخاص أو الشركات) المرتبطة بالشيكات.

### 6.2 إضافة جهة جديدة

1. اضغط على **"إضافة جهة جديدة"**.
2. أدخل:
   - **اسم الجهة** (إلزامي — حد أقصى 100 حرف)
   - **رقم الهاتف** (اختياري)
   - **ملاحظات** (اختياري — حد أقصى 500 حرف)
3. اضغط **"حفظ"**.

### 6.3 تعديل جهة

اضغط على **"تعديل"** في صف الجهة المطلوبة.

### 6.4 حذف جهة

اضغط على **"حذف"** — لا يمكن حذف جهة مرتبطة بشيكات موجودة.

## 7. إدارة المستخدمين (Admin فقط)

### 7.1 شاشة المستخدمين

تتيح لك إدارة حسابات المستخدمين الذين يمكنهم الدخول إلى النظام.

### 7.2 إضافة مستخدم جديد

1. اضغط على **إضافة مستخدم جديد**.
2. أدخل:
   - **اسم المستخدم** (إلزامي — فريد — حد أقصى 50 حرف — يستخدم لتسجيل الدخول)
   - **الاسم المعروض** (إلزامي — حد أقصى 100 حرف)
   - **كلمة المرور** (إلزامي عند الإضافة — 6 أحرف على الأقل)
   - **الدور**: ADMIN أو USER
3. اضغط **"حفظ"**.

### 7.3 تعديل مستخدم

1. اضغط على **"تعديل"** في صف المستخدم المطلوب.
2. يمكنك تغيير:
   - اسم المستخدم
   - الاسم المعروض
   - الدور
   - كلمة المرور (اتركها فارغة إذا لم ترد تغييرها)
3. اضغط **"حفظ"**.

> ⛔ لا يمكنك تغيير دورك من ADMIN إلى USER (حماية ذاتية).

### 7.4 تفعيل / تعطيل مستخدم

1. اضغط على زر **"تعطيل"** أو **"تفعيل"** في صف المستخدم.
2. المستخدم المُعطّل لا يمكنه تسجيل الدخول إلى النظام.

> ⛔ لا يمكنك تعطيل حسابك الخاص.

### 7.5 أعمدة الجدول

| العمود | الوصف |
|---|---|
| اسم المستخدم | اسم المستخدم المستخدم في تسجيل الدخول |
| الاسم المعروض | الاسم الكامل |
| الدور | ADMIN أو USER |
| الحالة | نشط / معطّل — مع شارة ملونة |
| الشيكات | عدد الشيكات المسجلة بواسطة هذا المستخدم |
| الإجراءات | أزرار تعديل / تفعيل-تعطيل |

## 9. حالات الشيك ودورة الحياة

### 9.1 الحالات

| الحالة (بالعربية) | الحالة (بالإنجليزية) | الوصف |
|---|---|---|
| مسجل | REGISTERED | تم تسجيل الشيك في النظام (الحالة الابتدائية) |
| مستلم/مسلّم | IN_HAND | تم استلام الشيك (للوارد) أو تسليمه (للصادر) |
| مصرف | CASHED | تم صرف الشيك — **حالة نهائية** |
| مرتجع | RETURNED | تم إرجاع الشيك (لعدم كفاية الرصيد أو أي سبب آخر) |
| ملغي | CANCELLED | تم إلغاء الشيك — **حالة نهائية** |

### 9.2 دورة الحياة

```
                     ┌─────────┐
                     │ مسجل    │
                     │REGISTERED│
                     └────┬────┘
                    ╱           ╲
                   ╱             ╲
            ┌──────────┐     ┌──────────┐
            │مستلم/مسلّم│     │  ملغي   │ ← حالة نهائية
            │ IN_HAND  │     │CANCELLED │
            └────┬─────┘     └──────────┘
              ╱   │    ╲
             ╱    │     ╲
      ┌────────┐ │  ┌──────────┐
      │ مصرف  │ │  │ مرتجع    │
      │CASHED │ │  │RETURNED  │
      └────────┘ │  └────┬─────┘
      حالة نهائية│      │
                 │   ╱   │    ╲
                 │  ╱    │     ╲
            ┌──────────┐ │  ┌──────────┐
            │مستلم/مسلّم│ │  │  ملغي   │
            │ IN_HAND  │ │  │CANCELLED │
            └──────────┘ │  └──────────┘
                         │  ┌──────────┐
                         │  │  مصرف   │
                         │  │ CASHED  │
                         │  └──────────┘
```

## 10. شرح رسائل الخطأ

### 10.1 رسائل تسجيل الدخول

| الرسالة | السبب | الحل |
|---|---|---|
| "اسم المستخدم أو كلمة المرور غير صحيحة" | اسم مستخدم غير موجود أو كلمة مرور خاطئة، أو الحساب مُعطّل | تحقق من البيانات أو تواصل مع Admin |
| "يرجى إدخال اسم المستخدم وكلمة المرور" | لم تدخل اسم المستخدم أو كلمة المرور | املأ جميع الحقول |

### 10.2 رسائل الشيكات

| الرسالة | السبب | الحل |
|---|---|---|
| "رقم الشيك مطلوب" | لم تدخل رقم الشيك | أدخل رقم الشيك |
| "المبلغ يجب أن يكون أكبر من 0" | المبلغ صفر أو سالب أو غير رقمي | أدخل مبلغًا صحيحًا |
| "تاريخ الاستحقاق يجب أن يكون أكبر من أو يساوي تاريخ الإصدار" | تاريخ الاستحقاق قبل تاريخ الإصدار | اختر تاريخًا مناسبًا |
| "لا يمكن تعديل شيك في حالة نهائية" | تحاول تعديل شيك مصرف أو ملغي | لا يمكن — الحالة نهائية |
| "لا يمكن حذف شيك في حالة نهائية" | تحاول حذف شيك مصرف أو ملغي | لا يمكن — الحالة نهائية |
| "لا يمكن تغيير الحالة إلى الحالة المطلوبة" | محاولة انتقال غير مسموح به | راجع دورة الحياة المسموحة |
| "البنك المحدد غير موجود" | البنك ربما حُذف أثناء استخدامك للنافذة | أعد تحميل الصفحة |
| "الجهة المحددة غير موجودة" | الجهة ربما حُذفت أثناء استخدامك للنافذة | أعد تحميل الصفحة |
| "حدث خطأ أثناء حفظ البيانات. حاول مرة أخرى." | خطأ تقني مؤقت | حاول مرة أخرى |

## 11. أسئلة شائعة

### س: هل يمكنني استرجاع شيك محذوف؟
لا. الحذف نهائي. يمكنك بدلًا من ذلك تغيير حالة الشيك إلى "ملغي" (CANCELLED) إذا كنت تريد إلغاءه دون حذفه.

### س: ماذا يعني "مستحق قريبًا"؟
يعني الشيكات التي تاريخ استحقاقها خلال 7 أيام القادمة، وغير المصرفة والملغية.

### س: هل يمكن لموظف عادي حذف شيك؟
لا. فقط المدير (ADMIN) يمكنه حذف الشيكات (غير النهائية).

### س: ماذا يحدث إذا حاولت دخول صفحة البنوك ولست Admin؟
سيتم إعادة توجيهك إلى الصفحة الرئيسية تلقائيًا.

### س: هل يمكنني تغيير كلمة مروري؟
حاليًا، يمكن لـ Admin تغيير كلمة المرور لأي مستخدم عبر شاشة تعديل المستخدم.

### س: هل أحتاج اتصال إنترنت لاستخدام النظام؟
لا، التطبيق يعمل محليًا بالكامل.

### س: ماذا لو انتهت صلاحية جلستي (Session)؟
سيتم تسجيل خروجك تلقائيًا بعد 24 ساعة وستظهر شاشة تسجيل الدخول.

## 12. نصائح سريعة

1. **استخدم الفلاتر** لتضييق نطاق البحث — خاصة عند وجود مئات الشيكات.
2. **البحث النصي** يبحث في رقم الشيك واسم الجهة.
3. **تاريخ الاستحقاق** هو التاريخ الأهم — ركز عليه في المتابعة اليومية.
4. **بطاقة "مستحق قريبًا"** تساعدك على معرفة الشيكات التي تحتاج متابعة عاجلة.
5. **الطباعة** تطبع فقط ما تراه في الجدول بعد تطبيق الفلاتر.
6. **إذا أردت إلغاء شيك بدل حذفه** — استخدم تغيير الحالة إلى "ملغي".
7. **تأكد من إضافة البنوك والجهات أولًا** قبل إضافة الشيكات.
8. **Admin واحد كافٍ** في النظام — يمكن تفعيل/تعطيل مستخدمين إضافيين حسب الحاجة.
