## المستخدمون والأدوار والصلاحيات

---

### 1. أنواع المستخدمين

في النسخة الأولى، النظام يدعم **نوعين فقط** من المستخدمين:

| النوع | الوصف |
|---|---|
| **Admin** (مشرف) | مدير النظام — يدير الشيكات والبيانات المرجعية والمستخدمين |
| **User** (مستخدم) | موظف إداري — يعمل على الشيكات فقط: عرض، إضافة، تعديل، تغيير حالة، طباعة |

> ملاحظة: يمكن إضافة دور "مدقق" أو "محاسب" في المرحلة الثانية عند الحاجة.

### 3. قواعد الوصول للبيانات

| القاعدة | الشرح |
|---|---|
| كل مستخدم يرى جميع البيانات | لا يوجد عزل للبيانات — كل الشيكات مرئية للجميع (مناسب لجهة واحدة) |
| المستخدم العادي لا يرى صفحة إدارة المستخدمين | قائمة المستخدمين وإضافة/تعديل/حذف المستخدمين متاحة للمشرف فقط |
| المستخدم العادي لا يدير البنوك والجهات | لتبسيط الـ MVP ومنع تكرار البيانات، إدارة البنوك والجهات تكون للمشرف فقط |
| حذف الشيكات | مقصور على المشرف فقط |
| حذف البنوك والجهات | مقصور على المشرف فقط |
| تعديل شيك في حالة نهائية | ممنوع للجميع (مصرف/ملغي) — قاعدة عمل وليست صلاحية أدوار |
| طباعة الكشف | متاحة للجميع |

---

### 4. قواعد إضافية

- **تسجيل الدخول:**
  - جميع المستخدمين يحتاجون إلى تسجيل دخول (Login/Password).
  - لا يوجد تسجيل عام (Self-registration) — المشرف فقط هو من يضيف مستخدمين جدد.
- **صلاحية التعديل على الشيكات:**
  - يمكن لأي مستخدم تعديل شيك طالما ليس في حالة "مصرف" أو "ملغي".
  - هذا مبني على قاعدة عمل وليس على صلاحية دور.
- **صلاحية تغيير الحالة:**
  - متاحة للجميع — لأن متابعة الحالة هي وظيفة أساسية لكل من يدخل الشيكات.
- **إدارة البيانات المرجعية:**
  - البنوك والجهات تُدار بواسطة Admin فقط في النسخة الأولى.
  - إذا احتاج المستخدم العادي بنكًا أو جهة غير موجودة، يطلب إضافتها من Admin.
